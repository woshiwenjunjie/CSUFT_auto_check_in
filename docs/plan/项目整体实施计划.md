## 阶段三：后端服务搭建

### 3.1 数据库设计（五张表）
| 表 | 用途 |
|----|------|
| `users` | 账号、密码哈希、加密 openid、打卡开关、延迟参数 |
| `configs` | 全局键值对配置 |
| `task_cache` | 缓存宿舍坐标等任务信息 |
| `checkin_records` | 打卡日期、坐标、距离、结果 |
| `operation_logs` | 执行详情日志，方便排查 |

### 3.2 API 开发
- `POST /auth/login` — 用户登录，返回 JWT
- `GET /tasks` — 获取任务列表
- `POST /checkin` — 手动触发打卡
- `GET /records` — 查询打卡记录
- `PUT /settings` — 更新用户设置

### 3.3 定时任务
- APScheduler cron 触发器，默认每天 21:00
- 每个用户执行前随机延迟（0~5 分钟），分散请求
- 失败自动重试机制

## 阶段四：部署上线

### 4.1 服务器准备
- 云服务器（推荐阿里云/华为云轻量）
- 安装 Python 3.11+、PostgreSQL、Nginx

### 4.2 部署链路
> Nginx（反向代理 + 限流）→ Supervisor（守护进程）→ FastAPI（uvicorn）→ PostgreSQL

### 4.3 一键部署脚本
- `scripts/deploy.sh` — 自动安装依赖、配置数据库、Nginx、Supervisor
- `scripts/update.sh` — git pull + 重装依赖 + 重启服务

## 阶段五：安全加固

- 密码：bcrypt 哈希存储 + MD5 哈希（学校接口要求）
- OpenID：AES-256-CBC 加密存储，密钥仅存环境变量
- 接口鉴权：JWT Token，过期重新登录
- 请求限流：Nginx 层限制认证接口频率
