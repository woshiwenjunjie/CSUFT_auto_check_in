# 配置向导

> `setup` 命令交互式配置详解

## 启动配置向导

```powershell
python scripts/cli.py setup
```

向导共 4 步，引导你完成全部首次配置。

## 第 1 步：输入 OpenID

OpenID 是微信小程序中你的唯一标识（`o` 开头，约 28 位）。

**选择捕获方式：**

| 方式 | 需 root？ | 说明 |
|------|----------|------|
| ✅ **模拟器自动脚本** | **不需要**（模拟器自带 root） | 一键命令，多账号切换方便 |
| `capture-openid` 自动捕获 | Android 7+ **需要 root** | 输入 `capture` 启动 mitmproxy |
| Fiddler 手动抓包 | Android 7+ **需要 root** | Windows 端代理工具 |
| Reqable 抓包 | **不需要 root** | 手机单机操作，推荐无 root 用户 |

**推荐：** 首次配置或有多账号需求，优先用 [模拟器自动脚本](../guides/user/模拟器自动抓取OpenID.md)：

```powershell
pip install mitmproxy
python scripts/tools/capture_openid_emulator.py --port 8899
```

- 如果尚未获取，输入 `capture` 自动启动 mitmproxy 捕获流程（需 root）
- 无 root 推荐用 Reqable：[Reqable 抓包指南](../guides/user/Reqable-抓包获取OpenID.md)
- 参考 [认证流程与抓包分析](../reference/认证流程与抓包分析.md) 了解各方案完整对比

**如何判断 OpenID 是否正确：** 格式为 `o` 开头，长度约 28 位字母数字组合。

## 第 2 步：输入学号

输入你的学号（如 `2023XXXXXX`）。

## 第 3 步：输入密码

- 密码不会在屏幕上显示（`getpass` 模式，显示 `*` 占位符）
- 可选择是否将密码保存到本地配置文件
- 密码以 `$obf:<base64>` 混淆存储，非明文
- 保存在 `~/.auto_check_in/config.json`，建议限制文件权限

## 第 4 步：自动验证登录

向导自动使用上一步的凭据调用学校 OAuth 接口：

- 验证 OpenID + 学号 + 密码是否有效
- 成功后获取 access_token 并保存
- 失败时仍保存凭据，方便后续排查

## 验证配置

```powershell
python scripts/cli.py status
```

确认输出包含：
- 登录状态：已登录
- 存在当前打卡任务
- 今日打卡记录（首次配置后应为空）

## 手动配置（高级）

```powershell
# 直接登录（跳过向导）
python scripts/cli.py login-openid <你的OpenID> <学号> --bind 1 --save-password

# 查看已保存的配置
python scripts/cli.py config

# 清除配置重新开始
python scripts/cli.py config clear --all
```
