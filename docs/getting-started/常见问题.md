# 常见问题

## 1. 打卡失败提示"签名错误"

先检查请求头是否完整，特别是 `Referer` 和 `User-Agent`。这两个头缺失时服务器会返回误导性的"签名错误"，实际签名算法可能没问题。

**排查步骤：**
1. 确认安装了最新版本（`git pull`）
2. 检查系统时间是否准确（时间偏差过大会导致签名失效）
3. 查看 `src/core/client.py` 中的环境头配置

## 2. Token 过期（返回 401）

access_token 有效期约 20~30 天，过期后 API 返回 `{"code": 401}`。

**解决：** 重新登录即可

```powershell
python scripts/cli.py login-openid --bind 0 --profile USER_1
```

OpenID 和学号会从配置自动读取，免密码（已绑定账号）。

## 3. GPS 超出范围

打卡时模拟坐标与宿舍距离超出精度上限。

**解决：**
- 使用 `--force` 参数强制提交（服务器端仍可能拒绝）
- 检查任务 ID 是否正确（`python scripts/cli.py tasks` 重新获取）
- 减小 `--offset` 值（如 `--offset 0.0001`）

## 4. 时区问题

API 时间基于 UTC，打卡窗口为 UTC 13:00–14:30（北京时间 21:00–22:30）。

- CLI 显示做 UTC+8 转换
- GitHub Actions Cron 使用 UTC 配置（`5 13 * * *`）
- SCF Cron 使用北京时间直接配置（`0 5 21 * * ? *`）

## 5. 已绑定错误（bindState）

```json
{"code": 400, "msg": "已绑定，请重新授权"}
```

**解决：** 使用 `--bind 0` 重新登录

```powershell
python scripts/cli.py login-openid --bind 0 --profile USER_1
```

## 6. 多账号管理

### 如何添加第二个账号？

```powershell
# 1. 捕获 OpenID（切换微信登录后重新运行）
python scripts/tools/capture_openid_emulator.py --port 8899

# 2. 创建新 profile
python scripts/cli.py config profile USER_2

# 3. 登录绑定
python scripts/cli.py login-openid --bind 0 --profile USER_2

# 4. 获取任务
python scripts/cli.py tasks --profile USER_2
```

### 如何批量打卡？

```powershell
# 默认打卡全部账号
python scripts/cli.py checkin

# 或指定部分账号
python scripts/cli.py checkin --profiles USER_1,USER_3
```

### 如何查看所有账号？

```powershell
python scripts/cli.py config profile list
```

## 7. 为什么必须用 OpenID？

小程序源码分析确认 `signIn` 和 `getCaptcha` 函数从未被调用，是死代码。服务器返回 `"Unauthorized grant type: password"`，拒绝密码授权。必须通过抓包获取 OpenID。

## 8. 手机开热点时怎么抓包？

手机本身无法设代理，可用：
- **Reqable**（原 HTTPCanary）：本地 VPN 抓包，无需代理
- **USB 反向共享**：电脑开 Fiddler，手机 USB 连接电脑并开启 USB 网络共享，再设置代理

## 9. 如何验证签名算法？

```bash
node scripts/sign.js /api/test 1700000000000 test_token
python -m pytest tests/test_cross_validate.py -v
```
