## 3. 三层架构

### 第一层：CLI 工具（已完成）

**文件**：`scripts/cli.py`（~1350 行）

单文件、零外部依赖（除项目自身模块和 httpx），支持 10 个子命令：

| 命令 | 用途 |
|------|------|
| `setup` | 交互式首次配置向导 |
| `status` | 一键状态概览 |
| `config` | 管理本地配置 |
| `login-openid` | OpenID 登录 |
| `login` | 密码登录（备用） |
| `tasks` | 查看任务列表 |
| `detail` | 查看任务详情 |
| `checkin` | 一键打卡 |
| `record` | 查询今日打卡状态 |
| `month` | 月度记录查询 |

### 第二层：GitHub Actions（已完成）

**文件**：`.github/workflows/auto-checkin.yml` + `scripts/auto_checkin.sh`

- cron 触发：每天 21:05 北京时间
- 凭据管理：GitHub Secrets（AES-256 加密存储）
- 通知：Server酱（微信）+ Telegram（备用）+ GitHub 邮件（兜底）
- 保活：每日 commit 防止 60 天无活动停用
- 日志：Artifact 保留 7 天

### 第三层：FastAPI Web 后端（阶段三，未开始）

**文件**：`src/main.py`（仅脚手架）+ `src/api/`、`src/models/`、`src/services/`（空目录）

目标：多用户管理、定时任务后台、Web 控制面板。

## 4. 核心模块详解

### 4.1 ApiClient（`src/core/client.py`）

项目的心脏，一个有状态的 HTTP 客户端。

**类结构**：

```python
class ApiClient:
    # 类属性（可通过环境变量覆盖）
    BASE_URL = "https://simp.csuft.edu.cn"
    WX_APP_ID = "wx0e47c34c9982aa09"
    WX_VERSION = 7
    TIMEOUT = 30.0  # 秒

    # 实例属性
    token: Optional[str]      # access_token
    base_url: str             # API 基址
    _client: httpx.Client     # 底层 HTTP 客户端
    _referer: str             # 伪装 Referer

    # 8 个 API 端点方法
    get_captcha()             # 获取验证码
    sign_in(username, pw)     # 密码登录
    sign_in_openid(...)       # OpenID 登录
    get_task_list(page, size) # 任务列表
    get_task_detail(task_id)  # 任务详情
    get_one_record(...)       # 今日打卡记录
    stu_sign(data)            # 提交打卡
    stu_late_sign(data)       # 补签
    get_month_records(...)    # 月度记录
```

**关键设计**：

- **上下文管理器**：`with ApiClient() as c:` 自动管理连接生命周期
- **自动签名**：`_headers()` 方法自动生成 `Authorization` / `FlySource-Auth` / `FlySource-sign`
- **指数退避重试**：`_request()` 网络/解析错误自动重试（1s/2s/4s，最多 3 次），401 不重试
- **微信伪装**：自动添加 `Referer` / `User-Agent` / `charset` 头
- **SSL 兼容**：`verify=certifi.where()` 解决 Windows Python 证书缺失；`trust_env=False` 避免系统代理干扰

### 4.2 签名工具（`src/utils/sign.py`）

```python
generate_sign(path, timestamp, token) → str   # FlySource-sign
generate_basic_auth() → str                     # Basic Auth
```

详见 `docs/guides/dev/签名算法详解.md`。

### 4.3 加密工具（`src/utils/crypto.py`）

```python
md5(data: str) → str          # MD5 十六进制小写
b64_encode(data: bytes) → str # Base64 编码
b64_decode(data: str) → bytes # Base64 解码
```

薄封装层，隔离底层库（hashlib / base64）。如果学校更换哈希算法，只需改这里。

### 4.4 GPS 工具（`src/utils/geo.py`）

```python
haversine(lat1, lng1, lat2, lng2) → float     # 球面距离（米）
random_offset(lat, lng, offset, seed) → tuple  # GPS 随机偏移
```

- `haversine`：使用标准球面余弦公式，精度足够（~0.1% 误差）
- `random_offset`：在 ±offset 范围内随机偏移，保留 6 位小数。`seed` 参数支持测试复现

## 5. 数据流

### 5.1 CLI 打卡流程

```
用户: python scripts/cli.py checkin
  │
  ├─① 加载配置 (~/.auto_check_in/config.json)
  │   token, username, openid, task_id
  │
  ├─② 获取任务详情
  │   ApiClient.get_task_detail(task_id)
  │   → 宿舍坐标 (locationLat, locationLng)
  │   → 精度上限 (locationAccuracy)
  │
  ├─③ 计算模拟坐标
  │   random_offset(dorm_lat, dorm_lng, offset=0.0003)
  │   → 在宿舍 ±30m 范围内随机生成 GPS 坐标
  │
  ├─④ 距离校验
  │   haversine(sim_lat, sim_lng, dorm_lat, dorm_lng)
  │   → 距离 < 精度上限 → 继续
  │   → 距离 > 精度上限 → 警告（--force 可跳过）
  │
  ├─⑤ 提交打卡
  │   ApiClient.stu_sign(stu_data)
  │   stu_data 包含: taskId, 坐标, stuTaskId(MD5校验), 日期
  │
  └─⑥ 回查确认
      ApiClient.get_one_record(task_id)
      → 确认服务器记录的打卡状态
```

### 5.2 GitHub Actions 自动流程

```
Cron 触发 (21:05 北京时间)
  │
  ├─① 注入 Secrets → 写 config.json
  │   CHECKIN_OPENID, CHECKIN_USERNAME, CHECKIN_PASSWORD
  │
  ├─② pip install -r requirements.txt
  │
  ├─③ login-openid --bind 0
  │   → 用 OpenID 登录换取 access_token（每次全新）
  │
  ├─④ tasks → 获取 task_id
  │
  ├─⑤ checkin → 提交打卡
  │
  ├─⑥ 结果判断 → notify()
  │   ├─ Server酱 HTTP POST (--data-urlencode)
  │   ├─ Telegram Bot API
  │   └─ GitHub Step Summary (Markdown)
  │
  └─⑦ 保活 commit (.keepalive)
      → git push [skip ci]
```

### 5.3 签名生成流程

```
请求发起
  │
  ├─ _headers(url_path)
  │   │
  │   ├─ ts = datetime.now(BEIJING_TZ).timestamp() * 1000
  │   │    → 13 位毫秒时间戳（北京时间）
  │   │
  │   ├─ sign = generate_sign(url_path, ts, self.token)
  │   │   → MD5(path + "?sign=" + MD5(ts + token)) + "1." + Base64(ts)
  │   │
  │   ├─ auth = generate_basic_auth()
  │   │   → "Basic " + Base64(ClientId + ":" + ClientSecret)
  │   │
  │   └─ 返回 dict:
  │       Authorization: auth
  │       FlySource-Auth: self.token
  │       FlySource-sign: sign
  │       Referer: "https://servicewechat.com/wx{AppID}/{version}/..."
  │       User-Agent: "Mozilla/5.0 ... MicroMessenger ..."
  │
  └─ _request() 发起 HTTP 请求
```

## 7. 如何扩展

### 7.1 添加新的打卡 API

1. 在 `src/core/client.py` 的 `ApiClient` 类中添加新方法
2. 在 `scripts/cli.py` 中添加对应子命令
3. 如果新 API 需要新的签名方式，修改 `src/utils/sign.py`

### 7.2 支持新的通知渠道

1. 在 `scripts/auto_checkin.sh` 中添加 `send_xxx()` 函数
2. 在 `notify()` 函数中调用
3. 在 `.github/workflows/auto-checkin.yml` 中添加对应的 `env` 注入
4. 更新 `docs/guides/user/Server酱配置教程.md` 的"多通道通知"章节

### 7.3 适配另一所学校

1. 修改 `src/core/client.py` 中的默认 BaseURL 和凭据
2. 适配不同的 API 路径和参数格式
3. 修改签名算法（如果目标学校不使用飞源平台）
4. 更新 GitHub Actions Secrets 中的凭据配置

### 7.4 启动阶段三（Web 后端）

参见 `docs/plan/服务器部署规划.md` 和 `docs/plan/项目整体实施计划.md`。
