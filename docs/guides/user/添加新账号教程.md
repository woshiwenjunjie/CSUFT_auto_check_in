# 添加新账号教程

> 适用版本：v0.12.0+  
> 功能：为本工具添加第二个、第三个……账号，支持多用户批量打卡

---

## 1. 账号是什么？

本工具用 **profile** 管理多账号。每个 profile 是一个独立账号，包含：

- **OpenID** — 微信小程序身份标识（o 开头，约 28 位）
- **学号** — 你的学号
- **Token** — 登录后获取的访问凭证

profile 名为 `default` / `USER_2` / `USER_3` … 可通过 `--profile` 指定。

> **设计说明**：密码字段可用 `--save-password` 保存（`$obf:` 混淆，仅防肩窥，非加密）。

---

## 2. 添加新账号（4 步完成）

### 2.1 抓取 OpenID

```bash
# 推荐：模拟器自动抓取
python scripts/tools/capture_openid_emulator.py --profile USER_N

# 或：mitmproxy 手动抓取
python scripts/cli.py capture-openid --profile USER_N
```

> 已有 OpenID 可跳过此步。查看 `~/.auto_check_in/config.json` 确认。

### 2.2 登录绑定

```bash
# OpenID + 学号都在命令行（新用户）
python scripts/cli.py login-openid oXXXX... 2023XXXXXX --profile USER_N

# OpenID 已在配置，只需提供学号（推荐）
python scripts/cli.py login-openid --profile USER_N --username 2023XXXXXX --bind 0
```

> `--username` 标志解决位置参数顺序问题，OpenID 在配置中时不必重复输入。

### 2.3 自动同步

登录成功后，系统自动同步到：

- **password.txt** — 更新 OpenID 和学号
- **deploy/tencent-scf/scf_env.json** — 重新生成 SCF 环境变量

无需手动操作。输出类似：

```
  ✓  同步配置到 password.txt ...
  ✓  password.txt 已更新
  ✓  重新生成 scf_env.json ...
  ✓  scf_env.json 已生成 (12 个变量)
  ✓  同步完成

  ✓  登录成功 (免密码登录)
  学号: 202**160
  Token: eyJ...
```

### 2.4 验证

```bash
# 查看今日任务
python scripts/cli.py tasks --profile USER_N

# 批量打卡测试（推荐：全部账号一起打卡）
python scripts/cli.py checkin

# 或仅打卡新账号
python scripts/cli.py checkin --profiles USER_N
```

## 3. SCF 重新部署（可选）

如果使用了腾讯云 SCF 自动打卡：

```bash
# 重新打包部署（自动包含新账号信息）
cd deploy/tencent-scf
python deploy.py
```

SCF 环境变量已在步骤 2.3 自动生成，部署时无需额外操作。

## 4. 管理多账号

### 切换当前账号

```bash
python scripts/cli.py config profile USER_2
```

### 查看所有账号

```bash
python scripts/cli.py config profile list
```

### 查看某个账号的配置

```bash
python scripts/cli.py config --profile USER_2
```

### 批量打卡

```bash
# 打卡全部已配置的账号
python scripts/cli.py checkin

# 打卡指定账号
python scripts/cli.py checkin --profiles USER_1,USER_2,USER_3
```

## 5. 完整示例：添加 USER_4

```bash
# 1. 抓取 OpenID（已有则可跳过）
python scripts/tools/capture_openid_emulator.py --profile USER_4

# 2. 登录（免密码，OpenID 已在配置中）
python scripts/cli.py login-openid --profile USER_4 --username 20234160 --bind 0

# 3. ✓ 已自动同步

# 4. 验证
python scripts/cli.py checkin --profiles USER_4
```

## 常见问题

### Q: 如何为已有用户更新 OpenID？
A: 重新抓取 OpenID 后，用 `login-openid --profile USER_N --username 学号 --bind 0` 重新登录。新 OpenID 自动覆盖旧值。

### Q: 密码保存了但登录失败？
A: 用 `--force-input` 重新输入密码：
```bash
python scripts/cli.py login-openid --profile USER_N --username 学号 --force-input
```

### Q: 为什么添加完新用户还要同步？
A: v0.12.0 起不需要——`login-openid` 成功后自动同步到 password.txt 和 scf_env.json。

### Q: config sync 命令还能手动用吗？
A: 可以，任何时候想手动同步都行：
```bash
python scripts/cli.py config sync
```
