## 1. 项目概述

自动打卡系统，目标学校：**中南林业科技大学**（`simp.csuft.edu.cn`），使用飞源（flySource）统一认证平台。

### 核心原理

模拟微信小程序的网络请求，通过逆向还原的签名算法和接口调用链，实现命令行打卡。

### 登录流程

```
微信小程序 wx.login()
  → 服务端换取 OpenID (需抓包获取)
  → OAuth2 换取 access_token
  → 携带 token + 签名调用打卡接口
```

### 接口调用链

```
获取 OpenID → OAuth 登录 → 查任务列表 → 查任务详情（含宿舍坐标）
  → 模拟 GPS 偏移 → 计算距离校验 → 提交打卡
```

### 技术栈

- Python 3.14 + httpx
- 签名算法：MD5 + Base64
- CLI 单文件入口：`scripts/cli.py`

## 3. 获取 OpenID

OpenID 是整个链路的前置条件。由于学校后端获取 OpenID 的接口已失效，只能通过抓包从微信小程序客户端截获。

### 3.1 方案选择

| 方案 | 难度 | 前提条件 |
|------|------|----------|
| MuMu 模拟器 + Fiddler | ⭐⭐ | Windows PC |
| 真机 root + Fiddler/HTTP Toolkit | ⭐⭐⭐ | 手机已 root |
| 真机免 root | ⭐⭐⭐⭐⭐ | VirtualXposed 等虚拟环境 |

**推荐：MuMu 模拟器方案**（无需 root 真机，已验证可行）。

### 3.2 材料清单

- Windows PC（本项目运行环境）
- MuMu 模拟器 12（自带 root）
- Fiddler Classic（免费）
- 微信 APK（[weixin.qq.com](https://weixin.qq.com) 下载）

### 3.3 步骤

#### 第一步：Fiddler 配置

1. 下载安装 Fiddler Classic：https://www.telerik.com/download/fiddler
2. **Tools → Options → HTTPS**：勾选 Capture HTTPS CONNECTs + Decrypt HTTPS traffic，选 from all processes
3. **Tools → Options → Connections**：勾选 Allow remote computers to connect，端口默认 8888
4. 重启 Fiddler
5. 查看本机 IP：`ipconfig | findstr "IPv4"`（如 `192.168.x.x`）

#### 第二步：模拟器配置

1. MuMu 模拟器安装微信 APK，扫码登录
2. 模拟器 WiFi 设置代理：主机名 = PC 的 IP，端口 = 8888

也可以用 ADB 一键配置：

```powershell
# 连接 MuMu（默认端口 7555）
& "D:\Program Files\Netease\MuMu\nx_main\adb.exe" connect 127.0.0.1:7555

# 设置全局代理（替换 IP 为你的实际 IP）
& "D:\Program Files\Netease\MuMu\nx_main\adb.exe" -s 127.0.0.1:7555 shell settings put global http_proxy <你的IP>:8888
```

#### 第三步：安装 Fiddler 证书到系统区（关键）

模拟器 `/system` 只读，需用 bind mount 绕过：

```powershell
# 下载证书
curl -o FiddlerRoot.cer http://<你的IP>:8888/FiddlerRoot.cer

# 转换为 PEM 并计算 hash
openssl x509 -in FiddlerRoot.cer -inform DER -out FiddlerRoot.pem -outform PEM
$hash = openssl x509 -in FiddlerRoot.pem -subject_hash_old -noout

# ADB root + 推送
adb root
adb push FiddlerRoot.pem /sdcard/

# 创建证书副本 + bind mount
adb shell "
mkdir -p /data/local/tmp/cacerts &&
cp /system/etc/security/cacerts/* /data/local/tmp/cacerts/ &&
cp /sdcard/FiddlerRoot.pem /data/local/tmp/cacerts/${hash}.0 &&
chmod 644 /data/local/tmp/cacerts/${hash}.0 &&
mount --bind /data/local/tmp/cacerts /system/etc/security/cacerts
"
```

> ⚠️ bind mount 在模拟器重启后失效，需重新执行最后一条 `mount --bind` 命令。

#### 第四步：抓包

1. Fiddler 按 `Ctrl+X` 清空会话
2. 模拟器内**删除「平安打卡」小程序**（长按图标 → 删除）→ 重新打开
3. Fiddler 过滤输入：`simp.csuft.edu.cn`
4. 找到响应中带 `getOpenidByJsCode` 的请求
5. 响应体 `data` 字段即为 OpenID（以 `o` 开头，28 位左右）

### 3.4 保存 OpenID

拿到 OpenID 后用 CLI 绑定学号：

```powershell
.\.venv\Scripts\Activate.ps1
python scripts/cli.py login-openid <OpenID> <学号>
# 输入密码（不回显）
```

成功后 token 保存至 `~/.auto_check_in/config.json`。

## 4. CLI 工具使用

### 4.1 命令速查

| 命令 | 功能 | 示例 |
|------|------|------|
| `login-openid` | OpenID 登录/绑定 | `python scripts/cli.py login-openid <OpenID> <学号>` |
| `tasks` | 查看任务列表 | `python scripts/cli.py tasks` |
| `detail <id>` | 查看任务详情 | `python scripts/cli.py detail 1` |
| `checkin <id>` | 打卡签到 | `python scripts/cli.py checkin 1` |
| `record <id>` | 查看当天打卡状态 | `python scripts/cli.py record 1` |
| `month <id> <YYYY-mm>` | 按月查询 | `python scripts/cli.py month 1 2026-06` |
| `login` | 密码登录（备用） | `python scripts/cli.py login <学号>` |

### 4.2 checkin 命令详解

```powershell
python scripts/cli.py checkin <task_id> [选项]
```

| 选项 | 说明 | 默认值 |
|------|------|--------|
| `--lat <纬度>` | 手动指定纬度 | 自动模拟（宿舍坐标 + 随机偏移） |
| `--lng <经度>` | 手动指定经度 | 自动模拟 |
| `--offset <值>` | GPS 随机偏移量（度） | `0.0003`（约 30 米） |
| `--force` | 超出范围仍提交 | 关闭 |
| `--late-date <YYYY-mm-dd>` | 补签日期 | 当天 |
| `--file-id <文件ID>` | 上传照片的文件 ID | 无 |

### 4.3 典型操作流程

```powershell
# 1. 登录（首次）
python scripts/cli.py login-openid oEXAMPLE_xxx 2023****

# 2. 查看打卡任务
python scripts/cli.py tasks
# 输出:
#   [1] 晚点名打卡  21:00-23:00
#   [2] 晨检打卡    07:00-09:00

# 3. 查看任务详情（含宿舍坐标和精度上限）
python scripts/cli.py detail 1
# 输出:
#   任务: 晚点名打卡
#   时间: 21:00 - 23:00
#   坐标: (28.138420, 112.638150)
#   精度: 100m

# 4. 模拟打卡（自动 GPS 偏移）
python scripts/cli.py checkin 1
# 输出:
#   使用模拟坐标: (28.138601, 112.638319)
#   与宿舍距离: 23.5m (精度上限: 100m)
#   打卡成功！

# 5. 确认打卡状态
python scripts/cli.py record 1
# 输出:
#   日期: 2026-06-07  状态: 已打卡
```

## 5. GitHub Actions 自动打卡

以上 1–4 节是本地 CLI 操作。配置完成后，**推荐部署到 GitHub Actions 实现每天自动打卡**，无需开电脑。

### 5.1 为什么用 GitHub Actions

| 对比 | 本地 CLI | GitHub Actions |
|------|----------|----------------|
| 需要开电脑 | ✅ 每天手动 | ❌ 全自动 |
| 需要联网 | ✅ | ❌（云端执行） |
| 费用 | 免费 | 免费（2000 分钟/月） |
| 通知 | 终端输出 | 微信推送 |
| 维护 | 手动更新 | 自动拉取最新代码 |

### 5.2 5 分钟部署

**第一步**：Fork 项目到自己的 GitHub 仓库（设为私有）

**第二步**：在仓库 Settings → Secrets and variables → Actions → New repository secret，添加：

| Secret 名称 | 你的值 |
|-------------|--------|
| `CHECKIN_OPENID` | 第 3 节抓包获取的 OpenID |
| `CHECKIN_USERNAME` | 你的学号 |
| `CHECKIN_PASSWORD` | 学校系统密码 |
| `SERVERCHAN_KEY` | Server酱 SendKey（`SCT` 开头） |

> `CHECKIN_TASK_ID` 可留空，脚本会自动获取。

**第三步**：获取 Server酱 SendKey
- 打开 [sct.ftqq.com](https://sct.ftqq.com) 微信扫码
- 复制 SendKey（`SCT` 开头）
- 添加到 GitHub Secrets：`SERVERCHAN_KEY`

**第四步**：手动触发测试
- Actions → Auto Check-In → Run workflow → Run workflow
- 等待约 30 秒，查看日志确认 `[通知] Server酱 HTTP 200: {"code":0...}`
- 微信应收到"方糖"公众号推送

### 5.3 通知说明

**所有结果都会推送微信通知**，不会漏掉任何一次执行：

| 打卡结果 | 微信标题 |
|----------|----------|
| 打卡成功 | ✅ CSUFT 打卡成功 |
| 今日已打卡 | ⏰ CSUFT 已打卡 |
| 未到签到时间 | ⏳ CSUFT 未到签到时间 |
| GPS 超出范围 | ⚠️ CSUFT GPS 超出范围 |
| Token 过期 | ❌ CSUFT Token 过期 |
| 登录/网络失败 | ❌ CSUFT 打卡失败 |

每条通知包含：日期、状态、距宿舍距离、失败原因 + 解决办法。

### 5.4 收不到通知？

在 Actions 日志中搜索 `[通知]`，查看：

```
通知渠道:
  ✅ Server酱 (微信) 已配置          ← 确认 Key 已注入
[通知] 正在发送 Server酱 推送...
[通知] Server酱 HTTP 200: {"code":0,"message":"success"}   ← API 响应
```

- 如果显示"未配置"→ 检查 GitHub Secrets 中 `SERVERCHAN_KEY` 是否正确
- 如果 HTTP 状态码不是 200 → 查看 `code` 字段对照错误码
- 如果 API 返回成功但微信没收到 → 确认关注了公众号"方糖"

详细排查见 `docs/guides/dev/GitHub-Actions部署记录.md` 第 6–7 节。

## 6. 签名算法说明

### FlySource-sign 请求头

学校 API 每个请求都需要携带签名，算法如下：

```
sign = MD5(path + "?sign=" + MD5(timestamp + token)) + "1." + Base64(timestamp)
```

参数说明：

| 参数 | 来源 | 说明 |
|------|------|------|
| `path` | 请求路径 | URL 去掉 ?query 部分，如 `/api/.../getListForApp` |
| `timestamp` | `new Date().getTime()` | 13 位 Unix 毫秒时间戳 |
| `token` | OAuth 响应 | `access_token`（JWT 格式） |
| `MD5` | 标准 MD5 | 32 位十六进制小写 |
| `Base64` | 标准 Base64 | 对时间戳字符串的 UTF-8 编码 |

### 请求头示例

```
Authorization: Basic <Base64(ClientId:ClientSecret)>
Content-Type: application/json
FlySource-Auth: <access_token>
FlySource-sign: <32位MD5>1.<Base64时间戳>
```

### stuTaskId 校验字段

打卡提交时的附加校验字段：

```
stuTaskId = MD5(JSON.stringify({
    latitude: ...,
    longitude: ...,
    locationAccuracy: ...,
    signDate: ...,
    taskId: ...,
    fileId: ...
}))
```

Key 顺序必须严格一致，与小程序源码中的对象字面量顺序相同。

### 固定凭据

| 属性 | 值 |
|------|-----|
| ClientId | `flysource_wise_wxapp` |
| ClientSecret | `DA788asdUDjnasd_flysource_wxappdsdadDAIUiuwqe` |
| BaseURL | `https://simp.csuft.edu.cn` |

> 凭据可通过环境变量覆盖：`FLYSOURCE_CLIENT_ID`、`FLYSOURCE_CLIENT_SECRET`、`CHECKIN_BASE_URL`

## 7. 安全设计

| 层级 | 措施 |
|------|------|
| 传输 | 密码做 MD5 后传输（学校接口要求） |
| 输入 | 密码交互式输入（`getpass`，不回显） |
| 存储 | 凭据文件位于 `~/.auto_check_in/config.json`，建议设权限 600 |
| 模拟 | GPS 随机小偏移（±0.0003 度 ≈ 30 米）、请求随机延迟 |
| 生产 | 密码 bcrypt、OpenID AES-256-CBC（阶段三实现） |

## 8. 常见问题

### Q: `[SSL: CERTIFICATE_VERIFY_FAILED]`

Windows Python 缺少系统证书。项目已内置 certifi 证书包解决。

### Q: 签名错误（code 400）

此问题已在 v0.7.0 彻底解决。根因是**服务器反爬校验**要求请求头包含微信小程序环境特征（`Referer`、`User-Agent`、`charset`），而非签名算法本身出错。

如果你遇到签名错误：
- 确认代码版本 ≥ v0.7.0
- 确认 `WX_APP_ID`/`WX_VERSION` 环境变量正确（默认值通常无需修改）
- 同步系统时间（`time.windows.com`）

### Q: OpenID 抓不到

- 确认小程序已删除后重新进入（强制触发完整登录流程）
- 确认手机/模拟器代理设置正确
- Android 7+ 需系统级证书（参考第 3 节 bind mount 方案）

### Q: 模拟器重启后抓包失效

bind mount 是临时的，重启后重新执行：

```powershell
adb root
adb shell mount --bind /data/local/tmp/cacerts /system/etc/security/cacerts
```

### Q: 距离校验不通过

- 降低 `--offset` 值（如 `--offset 0.0001`）
- 查看任务详情确认精度上限（`locationAccuracy`）
- 使用 `--force` 强制提交（风险自负）

### Q: GitHub Actions 执行了但微信没收到通知

1. 在 Actions 日志中搜索 `[通知]`
2. 如果显示"未配置"→ 在 GitHub Secrets 中添加 `SERVERCHAN_KEY`
3. 如果 API 返回 `code:40001` → SendKey 不正确，重新从 sct.ftqq.com 复制
4. 如果 API 返回 `{"code":0}` 但微信没收到 → 确认关注了公众号"方糖"
5. 如果日志没有任何 `[通知]` 输出 → 更新脚本到最新版本（v0.8.1+）

### Q: Telegram 怎么配置

1. 在 Telegram 找 `@BotFather`，发 `/newbot`，按提示创建 Bot，获得 Token
2. 给新 Bot 发任意一条消息
3. 浏览器访问 `https://api.telegram.org/bot<TOKEN>/getUpdates`，找到 `chat.id`
4. Token 添加到 GitHub Secrets → `TG_BOT_TOKEN`，chat.id → `TG_CHAT_ID`
