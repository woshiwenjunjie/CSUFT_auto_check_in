## 1. 认证相关

### OpenID

微信生态中，**用户 + 小程序**维度的唯一标识符。格式为 `o` 开头约 28 位字符串（如 `oABC123...`）。

- 同一微信用户在不同小程序中的 OpenID **不同**
- 同一用户在同一小程序中的 OpenID **不变**
- OpenID 通过微信 `wx.login()` 获取 js_code，再由服务端调用微信接口换取
- **本项目需要用户自行抓包获取 OpenID**，详见 `docs/guides/fiddler-抓包获取OpenID.md`

### access_token

OAuth2 认证中的**访问令牌**。用 OpenID + 学号 + 密码向学校认证服务器换取，后续所有打卡接口都需携带此令牌。

- 由飞源（flySource）认证服务颁发
- 有过期时间（通常 7-30 天），过期后需重新登录
- 在 HTTP 请求中以 `FlySource-Auth` 请求头传递
- CLI 存储在 `~/.auto_check_in/config.json`

### Basic Auth（Basic 认证）

HTTP 标准认证方式，格式为 `Basic base64(ClientId:ClientSecret)`。每个请求都携带，用于向服务器证明"这是一个合法的客户端应用"。

- **ClientId**: `flysource_wise_wxapp`（飞源微信小程序客户端标识）
- **ClientSecret**: 飞源为该客户端签发的密钥
- 不同于用户登录，Basic Auth 是**应用级别**的认证
- 在 HTTP 请求中以 `Authorization` 请求头传递

### tenant_id（租户 ID）

飞源平台中标识**学校/机构**的字段。中南林业科技大学的租户 ID 为 `000000`。

### grant_type

OAuth2 规范中的**授权类型**，决定登录方式：

| grant_type | 说明 | 对应 CLI 命令 |
|------------|------|-------------|
| `wxapp` | 微信小程序登录（用 OpenID） | `login-openid` |
| `password` | 密码登录（用学号+密码+验证码） | `login` |

### captcha（验证码）

密码登录时需要的图形验证码，由 `/api/flySource-auth/captcha` 接口生成。

- **无须 Basic Auth**（此端点在小程序中 `authorization=false`）
- 日常推荐用 OpenID 登录，可绕过验证码

## 2. API 请求相关

### FlySource-sign（签名）

飞源平台的自定义请求签名算法，用于**防止请求伪造和重放攻击**。每个请求都需携带。

**算法公式：**
```
sign = MD5(path + "?sign=" + MD5(timestamp + token)) + "1." + Base64(timestamp)
```

| 参数 | 说明 |
|------|------|
| `path` | 请求路径，去掉 query string（如 `/api/flySource-yxgl/dormSignTask/getListForApp`） |
| `timestamp` | 13 位毫秒时间戳（`Date.now()`） |
| `token` | access_token，未登录时为空字符串 |

**示例：**
```
输入: path="/api/test", timestamp=1700000000000, token="abc"
内层: MD5("1700000000000" + "abc")           → "3f8c..."
外层: MD5("/api/test?sign=3f8c...")          → "a1b2..."
结果: "a1b2..." + "1." + Base64("1700000000000")
```

JS 参考实现位于 `scripts/sign.js`，可用 `node scripts/sign.js <path> <ts> <token>` 交叉验证。

### stuTaskId

打卡提交时的一个**完整性校验哈希**，由打卡参数 JSON 序列化后 MD5 计算。

```
stuTaskId = MD5(JSON.stringify({
    latitude, longitude, locationAccuracy,
    signDate, taskId, fileId
}))
```

服务器端会用同样算法计算，比对是否一致，防止参数被篡改。

### Referer（请求来源）

HTTP 头，告诉服务器请求来自哪个页面。本项目伪装为微信小程序的 `servicewechat.com` 域：

```
Referer: https://servicewechat.com/wx0e47c34c9982aa09/7/page-frame.html
```

- `wx0e47c34c9982aa09` — 中南林平安打卡小程序的 AppID
- `7` — 小程序版本号

**如果缺少此头或值不正确，服务器会返回"签名错误"（误导性错误信息）。**

### User-Agent 伪装

请求头中伪装为微信内置浏览器的 UA 字符串（Android 微信环境），包含 `MicroMessenger`、`MiniProgramEnv/android` 等特征。缺少时服务器同样可能拒绝请求。

### 微信 AppID

`wx0e47c34c9982aa09` — 中南林业科技大学平安打卡小程序在微信平台的唯一标识。在 Referer 和签名上下文中使用。

## 3. 定位与打卡相关

### GPS 模拟偏移

学校打卡系统要求**定位在宿舍一定范围内**才算有效打卡。本项目用随机偏移模拟真实 GPS 定位：

```
模拟坐标 = 宿舍坐标 + random(-offset, +offset)
```

- 默认 `offset=0.0003`（约 33 米）
- 偏移后坐标保留 6 位小数（约 0.1 米精度）
- 可用 `--offset` 调整偏移量，或 `--lat/--lng` 手动指定

### Haversine 公式

**球面距离计算**公式，用于计算模拟坐标与宿舍坐标之间的距离，确保在精度上限内。

输入两个 GPS 坐标（经纬度），输出两点间球面距离（米）。实现位于 `src/utils/geo.py`。

### locationAccuracy（精度上限）

学校设定的**打卡有效范围**（米）。打卡时，模拟坐标与宿舍坐标的距离必须小于此值，否则服务器拒绝。

可用 `--force` 强制提交，但服务器端仍可能拒绝。

### signStatus（打卡状态码）

| 状态码 | 含义 | 终端颜色 |
|--------|------|---------|
| 0 | 已打卡 / 正常 | 🟢 绿色 |
| 1 | 迟到 | 🟡 黄色 |
| 2 | 请假中 | ⚪ 灰色 |
| 3 | 未归 | ⚪ 灰色 |
| 4 | 走读中 | ⚪ 灰色 |
| 5 | 离校中 | ⚪ 灰色 |
| 6 | 外宿中 | ⚪ 灰色 |

映射表定义在 `scripts/cli.py` 的 `STATUS_MAP` 常量中。

### 补签（late sign）

错过正常打卡时间窗口（通常 21:00-22:30）后，在特定条件下可以补签。使用 `stuLateSign` 接口，CLI 中通过 `--late-date YYYY-mm-dd` 触发。

### 打卡窗口

学校规定的每日打卡时间段，通常为 **21:00 — 22:30**。在此时间段外提交打卡会被拒绝。

## 4. 逆向分析相关

### wxapkg

微信小程序的**编译包体文件**，存储在小程序缓存目录中。用 `wxapp-unpacker` 等工具反编译可还原源代码。

本项目从 MuMu 模拟器中提取了 `_-250202245_7.wxapkg`，反编译后获得了签名算法、接口列表等关键信息。

### ADB（Android Debug Bridge）

安卓调试桥，用于与模拟器/手机通信。本项目用 ADB 获取 MuMu 模拟器的 root 权限并拉取小程序缓存文件。

```bash
adb connect 127.0.0.1:7555   # 连接 MuMu 模拟器
adb root                      # 获取 root 权限
adb pull /data/data/com.tencent.mm/MicroMsg/appbrand/pkg/ ./wxapkgs/
```

### MuMu 模拟器

网易出品的安卓模拟器（MuMu 12），用于运行微信并抓包。本项目用它运行中南林平安打卡小程序，配合 ADB 提取 wxapkg 包体。

### Fiddler

HTTP/HTTPS 抓包工具，用于拦截微信小程序与学校服务器之间的通信，获取 OpenID。

- 需安装根证书到模拟器才能解密 HTTPS 流量
- 过滤 `simp.csuft.edu.cn` 域名
- 登录时在请求中找到 `openid=o...` 参数

详见 `docs/guides/fiddler-抓包获取OpenID.md`。

## 5. 系统架构相关

### 飞源（flySource）

第三方统一认证平台，中南林业科技大学使用它管理**学生认证 + 打卡系统**。本项目中的所有 API 都是飞源平台提供的接口。

API 基址：`https://simp.csuft.edu.cn`

### config.json

CLI 的本地配置文件，位于 `~/.auto_check_in/config.json`，存储：

| 字段 | 说明 |
|------|------|
| `token` | access_token |
| `tenant_id` | 租户 ID（默认 000000） |
| `username` | 学号 |
| `openid` | 微信 OpenID |
| `password` | 密码（仅 `--save-password` 时保存） |
| `task_id` | 当前打卡任务 ID（`tasks` 命令自动保存） |

**所有凭据展示均经过 `_mask()` 脱敏处理。**

### SSL 证书问题（Windows）

Windows 上的 Python 可能缺少 SSL 根证书，导致 HTTPS 请求失败。本项目用 `certifi` 包提供证书，并在 `ApiClient` 中设置 `trust_env=False` 避免系统代理干扰。

### 请求重试机制

`ApiClient` 内置指数退避重试：遇到网络错误或 JSON 解析错误时自动重试（最多 3 次，间隔 1s / 2s / 4s）。401（未授权）不重试，直接返回。

### Token 过期

access_token 过期后，API 返回 `{"code": 401}` 或 msg 中包含 "过期" / "login" / "登录"。CLI 统一用 `_token_expired(resp)` 检测并提示重新登录。
