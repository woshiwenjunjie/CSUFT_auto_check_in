# 阶段三路线图

> FastAPI 后端服务搭建，实现多用户自动打卡管理

## 里程碑

| # | 节点 | 预计产出 |
|---|------|---------|
| M1 | 数据库设计完成 | 5 张表 DDL + SQLAlchemy ORM 模型 |
| M2 | 核心 API 可用 | 用户注册/登录、打卡、查询 API |
| M3 | 定时任务上线 | APScheduler 自动打卡 + Token 刷新 |
| M4 | 部署就绪 | Docker Compose / VPS 部署方案 + 监控 |

## 阶段 3.1：核心后端（1-2 周）

- FastAPI + SQLAlchemy 2.0 基础框架搭建
- 数据库表创建（users / tokens / task_cache / checkin_records / operation_logs / configs）
- 用户注册 API（`POST /api/v1/users/register`）
- 用户 CRUD（`GET/PATCH/DELETE /api/v1/users/{id}`）
- Token 手动刷新（`POST /api/v1/users/{id}/refresh-token`）
- 单用户手动打卡 API（`POST /api/v1/checkin/{user_id}`）
- 打卡状态查询（`GET /api/v1/checkin/{user_id}/status`）

## 阶段 3.2：定时任务（1 周）

- APScheduler 集成（`AsyncIOScheduler` + `SQLAlchemyJobStore`）
- 定时自动打卡逻辑（每天 21:00-21:05 触发）
- Token 临近过期自动刷新（剩余 < 3 天时）
- 打卡结果记录写入数据库
- 失败通知（Server酱 / 邮件 / Webhook）
- 操作日志记录

## 阶段 3.3：部署上线（1 周）

- Docker Compose 方案（FastAPI + PostgreSQL + Nginx）
- VPS 部署方案（systemd / Supervisor 守护）
- Nginx 反向代理 + SSL 证书
- 健康检查 endpoint（`GET /health`）
- 数据库备份策略（`pg_dump` + cron）
- 文档完善

## 阶段 3.4：管理后台（可选）

- Web UI（Vue/React）
- 用户管理界面
- 打卡记录可视化（月度日历图）
- 手动触发 + 日志查看

## API 设计概览

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/users/register` | 注册新用户 |
| GET | `/api/v1/users` | 用户列表 |
| PATCH | `/api/v1/users/{id}` | 更新用户 |
| DELETE | `/api/v1/users/{id}` | 删除用户 |
| POST | `/api/v1/users/{id}/refresh-token` | 刷新 token |
| POST | `/api/v1/users/{id}/sync-tasks` | 同步任务 |
| POST | `/api/v1/checkin/{user_id}` | 手动打卡 |
| GET | `/api/v1/checkin/{user_id}/status` | 查询今日状态 |
| GET | `/api/v1/checkin/{user_id}/month/{YYYY-mm}` | 月度记录 |
| POST | `/api/v1/schedules` | 创建定时任务 |
| PATCH | `/api/v1/schedules/{id}` | 修改打卡时间 |

## 数据库表

| 表 | 用途 |
|----|------|
| `users` | 账号、加密 openid、密码哈希、打卡开关 |
| `tokens` | access_token 生命周期管理 |
| `task_cache` | 缓存宿舍坐标等任务信息 |
| `checkin_records` | 打卡日期、坐标、距离、结果 |
| `operation_logs` | 执行详情日志 |
| `schedules` | 定时任务配置 |
| `configs` | 全局键值对配置 |
