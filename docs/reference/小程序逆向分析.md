## 1. 概述：微信小程序为什么可以逆向

微信小程序的包体格式为 .wxapkg，本质上是经过简单打包的 JavaScript 文件。与原生 Android APK 不同，wxapkg 的加密强度很低：

- **主体是 JS 源码**：小程序使用 uni-app 开发，编译产物为 minified JS，但变量名和逻辑结构保留
- **无代码混淆**：微信官方不强制混淆，大多数小程序（包括本目标）仅做压缩不做混淆
- **包体结构固定**：app-service.js（主逻辑，约 310KB）、app-config.json（配置）、页面文件等

这意味着只要有合适的反编译工具，完整还原小程序的业务逻辑是可行的。

### wxapkg 包体结构

```
xxx.wxapkg
+-- app-config.json        # 小程序全局配置（页面路径、window 样式等）
+-- app-service.js         # 主逻辑包，包含所有页面逻辑和公共模块（uni-app 编译产物）
+-- page-frame.html        # 页面框架模板
+-- pages/                 # 各页面的模板和样式（wxml/wxss）
|   +-- index/
|   +-- views/
|   +-- login/
+-- components/            # 组件文件
```

## 4. 核心逻辑还原

### 4.1 request.js -- 请求拦截器与签名算法

#### 全局配置常量

```javascript
// 从 app-service.js 提取
exports.BaseURL = "https://simp.csuft.edu.cn"
exports.ClientId = "flysource_wise_wxapp"
exports.ClientSecret = "DA788asdUDjnasd_flysource_wxappdsdadDAIUiuwqe"
exports.SignHeader = "FlySource-sign"
exports.TokenHeader = "FlySource-Auth"
```

ClientId 和 ClientSecret 是编译时写死的常量，不是动态获取的。每个学校有不同值，用于生成 Basic Auth 头。

#### MD5 与 Base64 实现

```javascript
// MD5 -- 使用微信内建 crypto 模块
function md5(s) {
    return crypto.createHash('md5').update(s, 'utf8').digest('hex')
}

// Base64 -- 微信基础库 gBase64 模块
gBase64.encode(str)
```

Python 等价实现：

```python
import hashlib, base64

def md5(data: str) -> str:
    return hashlib.md5(data.encode("utf-8")).hexdigest()

def b64_encode(data: bytes) -> str:
    return base64.b64encode(data).decode("utf-8")
```

#### 请求拦截器核心逻辑（完整还原）

```javascript
var interceptor = {
    invoke: function(config) {
        // 1. 获取当前毫秒时间戳
        var timestamp = (new Date).getTime()

        // 2. 读取已登录用户的 access_token
        var token = getToken()

        // 3. 拼接签名的 path（去掉 query string）
        var urlPath = config.url.split("?")[0]
        var signPath = urlPath + "?sign="

        // 4. 补全 BaseURL
        if (!config.url.startsWith("http")) {
            config.url = BaseURL + config.url
        }

        // 5. 超时 60s
        config.timeout = 60000

        // 6. 默认 Content-Type
        if (isEmpty(config.header)) {
            config.header = { "Content-Type": "application/json" }
        }

        // 7. Basic Auth（除非标记 authorization=false）
        if (config.authorization !== false) {
            var authRaw = ClientId + ":" + ClientSecret
            config.header["Authorization"] = "Basic " + gBase64.encode(authRaw)
        }

        // 8. 若有 token，注入 FlySource-Auth 和 FlySource-sign
        if (token) {
            config.header[TokenHeader] = token
            config.header[SignHeader] =
                md5(signPath + md5(timestamp + token))
                + "1."
                + gBase64.encode(timestamp.toString())
        }
    }
}

uni.addInterceptor("request", interceptor)
uni.addInterceptor("uploadFile", interceptor)
```

#### 签名公式图解

```
输入: path="/api/test", timestamp=1700000000000, token="abc123"

Step 1: signPath = "/api/test" + "?sign="
         -> "/api/test?sign="

Step 2: inner = md5("1700000000000" + "abc123")
         -> md5("1700000000000abc123")
         -> "3f8c6b9a..."

Step 3: outer = md5("/api/test?sign=" + "3f8c6b9a...")
         -> md5("/api/test?sign=3f8c6b9a...")
         -> "a1b2c3d4..."

Step 4: tsB64 = base64("1700000000000")
         -> "MTcwMDAwMDAwMDAwMA=="

Step 5: result = "a1b2c3d4..." + "1." + "MTcwMDAwMDAwMDAwMA=="
```

#### 请求出口函数

```javascript
exports.request = function(options) {
    return new Promise(function(resolve, reject) {
        uni.request(Object.assign({}, options, {
            success: function(res) {
                if (res.statusCode === 200) {
                    if (res.data.code === 401) {
                        toLoginPage()      // token 过期，跳转登录
                        reject(res.data)
                    } else {
                        resolve(res.data)
                    }
                } else {
                    reject(res)
                }
            },
            fail: reject
        }))
    })
}
```

code === 401 对应 Python 的 _token_expired(resp) 判断依据。

#### signIn -- 密码登录（死代码）

```javascript
signIn: function(tenantId, username, password, grantType, captchaKey, captchaCode) {
    return request({
        header: {
            "Content-Type": "application/x-www-form-urlencoded",
            "Tenant-Id": tenantId,
            "Captcha-Key": captchaKey,
            "Captcha-Code": captchaCode
        },
        url: "/api/flySource-auth/oauth/token",
        method: "POST",
        data: {
            tenantId: tenantId,
            username: username,
            password: md5(password),       // 密码先 MD5 再传输
            grant_type: grantType,         // "password"
            scope: "all"
        }
    })
}
```

#### signInOpenId -- OpenID 登录（实际使用的登录方式）

```javascript
signInOpenId: function(tenantId, username, password, openid, bindState) {
    return request({
        header: {
            "Content-Type": "application/x-www-form-urlencoded",
            "Web-Type": "wxapp",           // 声明小程序端
            "Tenant-Id": tenantId
        },
        url: "/api/flySource-auth/oauth/token",
        method: "POST",
        data: {
            tenantId: tenantId,
            username: username,
            password: password !== "" ? md5(password) : "",
            grant_type: "wxapp",           // 不同于密码登录的 "password"
            openid: openid,                // OpenID 是认证核心
            bindState: bindState,          // 0=仅登录 1=绑定并登录
            scope: "all"
        }
    })
}
```

bindState 的两种模式：

| bindState | 场景 | username/password | 服务器行为 |
|-----------|------|-------------------|-----------|
| 0 | 首页加载（仅登录） | 空字符串 | 用 OpenID 换取 token，不绑定学号 |
| 1 | 绑定页面 | 填入学号和密码 | 将 OpenID 与学号绑定，同时返回 token |

#### 完整登录流程（从源码还原）

```
首页 onLoad()
  | 检查 storage 中的 access_token
  |
  +- token 有效 -> 直接显示首页，加载任务列表
  |
  +- token 不存在 -> wx.login()
       | 获取临时 js_code
       |
       +-> getOpenidByJsCode(js_code)
       |    |
       |    +-> 响应: { data: openid }
       |
       +-> signInOpenId(openid, bindState=0)
       |    |
       |    +-> 已绑定用户: 返回 access_token -> 保存 -> 加载任务
       |    |
       |    +-> 未绑定用户: error("invalid_grant")
       |         +-> 弹窗提示 -> 跳转绑定页
       |
       +-> 绑定页: signInOpenId(openid, username, password, bindState=1)
            |
            +-> 成功: 保存 token + 用户信息 -> 返回首页
```

#### GPS 定位获取

```javascript
function getLocationAndCheck() {
    wx.getLocation({
        type: "gcj02",                    // 国测局坐标系（GCJ-02）
        isHighAccuracy: true,
        highAccuracyExpireTime: 3100,     // 3.1 秒缓存
        success: function(res) {
            this.stuSignData.signLat = res.latitude
            this.stuSignData.signLng = res.longitude

            // 计算与宿舍的距离
            this.stuSignData.locationAccuracy = getDistance(
                this.stuSignData.signLat, this.stuSignData.signLng,
                this.location.locationLat, this.location.locationLng
            )

            // 校验是否在精度范围内
            this.checkStuSign =
                Number(this.stuSignData.locationAccuracy)
                < Number(this.location.locationAccuracy)

            // 构造 stuTaskId
            var hashData = {
                latitude: res.latitude.toString(),
                longitude: res.longitude.toString(),
                locationAccuracy: this.stuSignData.locationAccuracy.toString(),
                signDate: this.signDate,
                taskId: this.params.taskId,
                fileId: ""
            }
            this.stuSignData.stuTaskId = md5(JSON.stringify(hashData))

            this.getLocation = new Date
        }
    })
}
```

#### 距离计算函数（getDistance）

```javascript
function getDistance(lat1, lng1, lat2, lng2) {
    var radLat1 = lat1 * Math.PI / 180
    var radLat2 = lat2 * Math.PI / 180
    var deltaLat = radLat1 - radLat2
    var deltaLng = lng1 * Math.PI / 180 - lng2 * Math.PI / 180
    var a = 2 * Math.asin(
        Math.sqrt(
            Math.pow(Math.sin(deltaLat / 2), 2)
            + Math.cos(radLat1) * Math.cos(radLat2)
            * Math.pow(Math.sin(deltaLng / 2), 2)
        )
    )
    a *= 6378.137
    return Math.round(10000 * a) / 10
}
```

Python 等价实现：

```python
import math

def haversine(lat1, lng1, lat2, lng2):
    R = 6371000
    phi1, phi2 = math.radians(lat1), math.radians(lat2)
    dphi = math.radians(lat2 - lat1)
    dlambda = math.radians(lng2 - lng1)
    a = math.sin(dphi/2)**2 + math.cos(phi1)*math.cos(phi2)*math.sin(dlambda/2)**2
    return R * 2 * math.atan2(math.sqrt(a), math.sqrt(1-a))
```

JS 原版用地球半径 6378.137km（赤道半径），Python 用 6371km（平均半径），差异约 0.1%，对打卡校验无影响。

## 6. 死代码分析

### 6.1 signIn -- 密码登录从未被调用

signIn() 函数虽然在小程序源码中完整存在，但从未被任何页面调用。所有登录走 signInOpenId() 路径。

源码证据：
- 在 app-service.js 中全文搜索 signIn(（带左括号的调用）-- 结果只有函数定义本身，没有调用点
- 全局搜索 grant_type=password -- 在请求体中从不出现，实际使用的是 grant_type=wxapp
- 服务器对 grant_type=password 的响应是 "Unauthorized grant type: password"

### 6.2 getCaptcha -- 验证码接口从未被调用

getCaptcha() 同样只有定义没有调用。验证码页面在 UI 中不存在，小程序首页直接触发 OpenID 登录。

### 6.3 死代码存在的原因

推测是飞源统一认证平台为兼容"密码登录"场景保留了代码路径，但 CSUFT 的部署配置仅启用了微信小程序登录（wxapp）。死代码的存在说明小程序是模板工程裁剪而来。
