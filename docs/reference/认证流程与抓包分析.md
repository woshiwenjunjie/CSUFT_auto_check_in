## 1. 认证全链路时序图

### 1.1 登录流程概览

小程序使用飞源统一认证 + 微信 OpenID 登录。逆向分析确认密码登录（signIn）从未被调用，所有登录走 OpenID 链路。

```
时间轴 ->

用户打开小程序
    |
    v
wx.login()
    | 生成临时 js_code（5 分钟有效）
    |
    v
GET /api/flySource-base/openApi/getOpenidByJsCode?jsCode=xxx
    |
    v
响应: { "data": "oABC123..." }        <- Key: OpenID 在这里
    |
    v
POST /api/flySource-auth/oauth/token
    | Content-Type: application/x-www-form-urlencoded
    | Authorization: Basic <客户端凭据>
    | Web-Type: wxapp
    | Tenant-Id: 000000
    |
    | 请求体:
    |   grant_type=wxapp
    |   &tenantId=000000
    |   &username=2023XXXXXX
    |   &password=e10adc39...          <- 密码的 MD5
    |   &openid=oABC123...             <- Key: OpenID 也在这里
    |   &bindState=1                   <- 0=仅登录 1=绑定并登录
    |   &scope=all
    |
    v
响应: { "access_token": "eyJh...", "expires_in": 2592000 }
    |                                   <- Key: Token（30 天有效）
    v
GET /api/flySource-yxgl/dormSignTask/getListForApp
    | FlySource-Auth: eyJh...          <- 携带 token
    | FlySource-sign: a1b2...          <- 携带签名
    |
    v
响应: { "data": { "records": [...] } } <- 打卡任务列表
```

### 1.2 认证核心要点

- **OpenID 是永久标识**：只要微信账号不解绑，OpenID 不变，CLI 配置一次即可重复使用
- **BindState 两种模式**：0 仅登录（首页自动触发），1 绑定并登录（用户首次配置时触发）
- **Token 有效期 30 天**：expires_in: 2592000，到期需用 OpenID 重新获取
- **服务器拒绝 password 授权**：明确返回 "Unauthorized grant type: password"

## 2. 四种抓包方案速查表

### 关键条件总览

| 方案 | 需 root？ | 需电脑？ | 需 WiFi？ | 原理 | 优点 | 缺点 |
|------|----------|---------|----------|------|------|------|
| ✅ **模拟器自动脚本** `capture_openid_emulator.py` | 不需要（模拟器自带 root）| 是（运行脚本）| 不需要 | ADB 代理 + mitmproxy + 自动提取 | 一键全自动，多账号切换方便，模拟器隔离安全 | 需电脑，需模拟器 |
| **mitmproxy** `capture-openid` | **Android 7+ 需要 root**（证书移系统区）；Android 6- 不需要 | 是（运行代理） | 同 WiFi | HTTP 代理 + 自动提取 | 一键自动化，无需手动翻日志 | 需 root 或旧安卓 |
| **Fiddler** | **Android 7+ 需要 root**（同上）；Android 6- 不需要 | 是（运行 Fiddler） | 同 WiFi | HTTP 代理 + 断点调试 | 功能丰富，支持修改请求 | 需 root 或旧安卓 |
| **Fiddler + 模拟器** | **不需要**（模拟器 Android 5/6 无此限制） | 是（运行模拟器） | 不需要 | 模拟器中抓包 | 无 root 限制 | 需电脑性能好，不能真机 |
| **Reqable** | **不需要**（VPN 抓包绕过证书限制） | 否（手机单机） | 不需要（热点可用） | 本地 VPN 拦截 | 手机单机，热点可用 | 仅 Android |
| **Reqable + ADB 证书** | **不需要 root**，但 Android 11+ 需一次 ADB 命令 | 需要一次电脑 ADB | 不需要 | VPN + 系统证书 | 无需 root，长期有效 | 需一次电脑连接 |

### 2026 年推荐策略

| 场景 | 推荐方案 |
|------|---------|
| **有多账号需求**（如本项目中 3 个账号） | ✅ **模拟器自动脚本** — 一键捕获，切换微信账号方便，模拟器隔离免风控 |
| 首次配置，有 PC | ✅ **模拟器自动脚本** 或 **Reqable**（ADB 装证书） |
| 手机开热点时 | ✅ **Reqable**（VPN 模式） |
| 已有 root 手机 | **Fiddler** 或 **mitmproxy** |

## 3. Fiddler 详细步骤

> **关键条件**：需电脑 + 同 WiFi + **Android 7+ 需 root**（或使用 Android 6- 模拟器）

### 3.1 下载安装

1. 访问 https://www.telerik.com/download/fiddler 下载 Fiddler Classic（免费版，不是 Fiddler Everywhere）
2. 安装：一路 Next -> I Agree -> Install -> Finish
3. 启动 Fiddler Classic

### 3.2 配置 HTTPS 解密

1. 菜单栏 Tools -> Options -> HTTPS 选项卡
2. 勾选：Capture HTTPS CONNECTs 和 Decrypt HTTPS traffic
3. 下拉选择 from all processes
4. 证书安全警告点击 Yes -> 再次 Yes（安装根证书）
5. 切到 Connections 选项卡
6. 勾选 Allow remote computers to connect
7. 端口默认 8888
8. OK 保存，重启 Fiddler

### 3.3 设置手机代理

1. 确保电脑和手机连接同一 WiFi
2. 查看电脑 IP：Fiddler 右上角 Online 指示器，或 ipconfig 命令
3. 手机 WiFi 设置：
   - Android：长按 WiFi -> 修改网络 -> 高级选项 -> 代理设为手动 -> 填入电脑 IP 和端口 8888
   - iOS：WiFi 右侧 (i) -> HTTP 代理 -> 配置代理 -> 手动 -> 填入 IP 和端口
4. 手机浏览器访问 http://<电脑IP>:8888，下载并安装 Fiddler 根证书
5. 验证：手机浏览器访问 HTTPS 网站，Fiddler 中出现解密后的请求记录

### 3.4 抓包获取 OpenID

1. Fiddler 中按 Ctrl + X 清空会话列表
2. 手机上删除微信小程序（首页下拉 -> 长按 -> 删除），确保触发完整登录流程
3. 手机上重新打开小程序
4. Fiddler 过滤器输入 simp.csuft.edu.cn
5. 查找 GET /api/flySource-base/openApi/getOpenidByJsCode?jsCode=xxx
6. 点击该请求 -> 右侧 Inspectors -> 下半部分 JSON 视图
7. 复制响应体中的 data 字段值（以 o 开头的字符串）

> 备选：如果 getOpenidByJsCode 未触发，查找 POST /api/flySource-auth/oauth/token，在请求体的 openid= 字段中获取。

### 3.5 安装 Fiddler 根证书到 Android

> **核心问题**：Android 7+ 系统不信任用户安装的 CA 证书，微信作为非调试应用（targetSdkVersion >= 24）拒绝使用用户级证书 → HTTPS 握手失败 → 小程序打不开 → 抓不到任何请求。

#### 方案 A：已 root 的手机（推荐）

用 Magisk 模块将证书移到系统信任区：
```bash
# 1. 导出 Fiddler 根证书（Tools -> Options -> HTTPS -> Actions -> Export Root Certificate to Desktop）
# 2. 用 OpenSSL 转为 Android 系统证书格式
openssl x509 -inform DER -in FiddlerRoot.cer -out FiddlerRoot.pem
openssl x509 -inform PEM -subject_hash_old -in FiddlerRoot.pem | head -1  # 得到 hash
# 3. 重命名并复制到系统证书目录
cp FiddlerRoot.pem /system/etc/security/cacerts/<hash>.0
chmod 644 /system/etc/security/cacerts/<hash>.0
```
或使用 Magisk 模块 Move Certificates 一键迁移。

#### 方案 B：无 root — 使用 Android 6- 模拟器

MuMu / 雷电模拟器可创建 Android 5/6 镜像，无此限制：
1. MuMu 设置 → 安卓版本 → 选 Android 6
2. 模拟器中安装微信和小程序
3. 在模拟器内设置 Fiddler 代理
4. Fiddler 中抓包

#### 方案 C：无 root — VirtualXposed

在 VirtualXposed 虚拟环境中运行微信，虚拟环境可信任用户证书。

#### 方案 D：无 root — 直接换 Reqable（推荐）

Reqable 的 VPN 模式不依赖代理 CA 证书，**不需要 root**。见下方第 4 节。

### 3.6 防火墙配置

如果手机无法连接 Fiddler，检查 Windows 防火墙：

```powershell
# 管理员权限 PowerShell 添加入站规则
New-NetFirewallRule -DisplayName "Fiddler" -Direction Inbound -Protocol TCP -LocalPort 8888 -Action Allow
```

## 4. Reqable 详细步骤

> **关键条件**：仅需手机（Android），**不需要 root**，不需要电脑，热点可用

### 4.1 下载安装

1. 手机浏览器访问 https://httpcanary.com（Reqable 原名 HTTPCanary，官网沿用旧名）
2. 下载 APK 并安装
3. 或从 Google Play 搜索 "HTTPCanary" 安装

### 4.2 安装 CA 证书

> **不需要 root**。Reqable 的 VPN 模式下不依赖代理式 CA 拦截，安装证书仅用于部分深度检测场景。

1. 打开 Reqable
2. 左上角菜单 -> 设置 -> SSL 证书设置 -> 安装 CA 证书到系统
3. 按提示操作：
   - Android 7-10：通常可直接安装
   - Android 11+：系统限制了用户安装证书到系统区，但 Reqable 内置了**免 root 方案**：点击"移动证书"后，Reqable 给出 ADB 命令（仅需一次，后续不再需要电脑）
4. 证书安装成功后返回 Reqable

### 4.3 开始抓包

1. 回到 Reqable 主界面
2. 点击底部"开始捕获"按钮（红色圆点）
3. Reqable 弹出 VPN 连接请求 -> 点确定
4. 状态栏出现钥匙图标表示抓包已启动

### 4.4 打开打卡小程序

1. 不要关闭 Reqable，直接按 Home 键回到桌面
2. 打开微信
3. 进入 CSUFT 平安打卡 小程序
4. 正常使用小程序，让它完成登录流程（加载首页即可）
5. 切回 Reqable

### 4.5 查找 OpenID

1. 点击底部停止捕获按钮
2. 在列表顶部搜索栏输入：getOpenidByJsCode
3. 点开搜索结果中唯一的那条请求
4. 切换到响应体（Response Body）标签
5. 找到 "data": "o..." -- 那个 o 开头的字符串就是 OpenID

### 4.6 使用 OpenID

```powershell
python scripts/cli.py setup
```

按提示输入 OpenID + 学号 + 密码，自动完成登录和配置。或手动：

```powershell
python scripts/cli.py login-openid <OpenID> <学号>
```

## 5. mitmproxy 自动捕获

> **关键条件**：需电脑 + 同 WiFi + **Android 7+ 需 root**（或使用 Android 6- 设备/模拟器）

### 5.0 推荐方式：模拟器自动捕获（免 root，安全隔离）

如果不想折腾真机 root，`capture_openid_emulator.py` 脚本在 Android 模拟器中自动化整个过程：

```bash
# 前提：模拟器已运行 + ADB 可连接 + 微信已安装
python scripts/tools/capture_openid_emulator.py
```

脚本会自动：
1. 检查 ADB 连接、mitmproxy 环境
2. 安装 CA 证书到模拟器（有 root 的模拟器自动推系统区，无 root 的推 /sdcard 提示手动装）
3. 设置模拟器 HTTP 代理指向本机
4. 启动 mitmproxy + capture_addon
5. 通过 ADB 打开微信小程序
6. 捕获到 OpenID 后自动清理代理并退出

各模拟器兼容性：

| 模拟器 | 证书安装 | 用户操作 |
|--------|---------|---------|
| AVD (adb root 可用) | 自动推系统区 | 无需操作 |
| MuMu 12（自带 root） | bind mount 到系统区 | 无需操作 |
| MuMu 安卓 6 镜像 | 推 /sdcard 手动装 | 设置 → 安全 → 安装证书 |
| 雷电 安卓 5 镜像 | 推 /sdcard 手动装 | 同上 |
| Genymotion | 自动推系统区 | 无需操作 |

详细用法：`python scripts/tools/capture_openid_emulator.py --help`

#### 关于安全性的分析：为什么模拟器先 root 再登录微信是合理的

**核心问题**：使用 mitmproxy 截获微信小程序的 HTTPS 流量，是否会触发微信的风控导致账号被封？

**技术原理**：
- mitmproxy 对小程序→学校 API 流量做 TLS 中间人（MITM）
- Android 7+ 只信任系统级 CA 证书，必须将 mitmproxy CA 推入系统区（通过 root 权限）
- 微信内部通信（如 wx.login）使用 SSL Pinning，不受系统证书影响
- 被拦截的仅是小程序向 `simp.csuft.edu.cn` 发起的明文 API 调用

**先 root 模拟器 → 注入系统证书 → 再登录微信 这个顺序更安全吗？**

是的，原因有 3 层：

| 维度 | 先登录再装证书 | 先装证书再登录 | 说明 |
|------|--------------|--------------|------|
| 信任链一致性 | 系统信任存储中途变化 | 系统环境始终一致 | 证书一致性降低异常检测特征 |
| 证书指纹校验 | 首次握手在校验前 | 证书在首次握手前已就位 | 微信 SSL Pinning 校验时环境一致 |
| 行为基线建立 | 无证书→有证书切换 | 始终有证书 | 避免"信任存储变更"这类异常信号 |

**关键事实**：微信小程序与学校 API 之间的 TLS 连接**不使用 SSL Pinning**（这是它能被抓包的前提）。微信核心通信（登录、支付等）使用独立的 SSL Pinning 通道，不受模拟器中系统证书影响。

**实际验证**：v0.11.0 通过模拟器自动脚本完成 3 个账号的 OpenID 捕获，均先装证书再登录微信，无一触发风控。

**结论**：在模拟器中用 mitmproxy 捕获 OpenID 是低风险操作：
- 被拦截的 API 不涉及微信核心通信
- 模拟器隔离了真实设备环境
- 先装证书再登录微信的顺序进一步降低了异常检测风险
- 即使模拟器被风控标记，重置镜像即可，微信账号本身不受影响

### 5.1 原理（mitmproxy）

mitmproxy 是一个支持 Python 脚本扩展的中间人代理工具。通过编写 addon 脚本，可以监听 HTTP 响应，当检测到 getOpenidByJsCode 的响应时自动提取 OpenID 并保存到配置文件。

### 5.2 安装 mitmproxy

```powershell
pip install mitmproxy
```

### 5.3 capture-openid 一键命令

项目 CLI 提供了封装命令，免去手动启动 mitmproxy 的繁琐：

```powershell
python scripts/cli.py capture-openid
```

该命令会自动：
1. 启动 mitmdump 并使用 scripts/capture_addon.py 插件
2. 在端口 8080 监听
3. 手机设置代理到电脑的 8080 端口
4. 打开小程序后自动捕获 OpenID
5. 将 OpenID 写入 ~/.auto_check_in/config.json
6. 自动关闭 mitmproxy

### 5.4 插件代码解读

capture_addon.py 的核心逻辑：

```python
import json
from pathlib import Path
from mitmproxy import http, ctx

CONFIG_FILE = Path.home() / ".auto_check_in" / "config.json"

class OpenIDCapture:
    def __init__(self):
        self.captured = False

    def response(self, flow: http.HTTPFlow):
        if self.captured:
            return
        if "getOpenidByJsCode" not in flow.request.pretty_url:
            return

        try:
            data = flow.response.json()
            openid = data.get("data", "")
            if not openid or not openid.startswith("o"):
                return

            self.captured = True

            cfg = {}
            if CONFIG_FILE.exists():
                cfg = json.loads(CONFIG_FILE.read_text(encoding="utf-8"))
            cfg["openid"] = openid
            CONFIG_FILE.parent.mkdir(parents=True, exist_ok=True)
            CONFIG_FILE.write_text(
                json.dumps(cfg, ensure_ascii=False, indent=2), encoding="utf-8"
            )

            print(f"OpenID 已捕获并保存到 {CONFIG_FILE}")
            ctx.master.shutdown()
        except Exception as e:
            print(f"[capture_addon] 解析响应失败: {e}")

addons = [OpenIDCapture()]
```

### 5.7 root 要求说明

mitmproxy 与 Fiddler 原理相同——都是**中间人代理**，需要手机信任代理的 CA 证书。

**核心问题**：微信的 `targetSdkVersion >= 24`（对应 Android 7），这意味着：
- 微信只信任**系统级 CA 证书**
- 用户安装的 CA 证书（mitm.it 下载安装的）**不被信任**
- HTTPS 握手失败 → 微信显示"网络不给力" → 抓不到任何请求

**解决方案**（同 Fiddler 3.5 节）：

| 方案 | 条件 | 说明 |
|------|------|------|
| root 手机 + 证书移系统区 | 需 root | Magisk 模块一键迁移，一劳永逸 |
| 换用 Android 6- 模拟器 | 无需 root | MuMu/雷电创建 Android 6 镜像 |
| 换用 Reqable | 无需 root | VPN 模式，不依赖代理 CA 证书（推荐） |

## 6. 网络环境策略

### 场景一：电脑和手机在同一 WiFi（最常用）

适用方案：Fiddler 或 mitmproxy

操作要点：
1. 电脑连 WiFi，手机连同一 WiFi
2. 确保 WiFi 的 AP 隔离（AP Isolation）功能关闭（部分公共 WiFi 有此限制）
3. 防火墙放行代理端口（8888 / 8080）

### 场景二：手机开热点，电脑连手机热点

适用方案：Reqable

手机开热点时手机本身无法设代理（手机是网关，无法代理自身流量），因此 Fiddler/mitmproxy 不可用。Reqable 的 VPN 模式可以绕过此限制。

### 场景三：USB 反向网络共享

当没有 WiFi 环境时，可以使用 USB 共享：

1. 电脑开启代理（Fiddler / mitmproxy）
2. 手机通过 USB 连接电脑
3. 手机开启 USB 网络共享（设置 -> 个人热点 -> USB 网络共享）
4. 手机 WiFi 代理指向电脑 IP（此时 USB 共享的网络接口）

注意：此方案在不同安卓 ROM 上表现不一，部分机型不支持 USB 共享时修改代理。

### 场景四：热点 + USB 共享混合

手机开热点供电脑上网，同时电脑通过 USB 连接手机，用 Fiddler 代理手机流量。此方案较为复杂但功能最完整。

### 方案选择总结

| 网络环境 | 推荐方案 | 需 root？ | 理由 |
|---------|---------|----------|------|
| 同 WiFi + 有 root | Fiddler 或 mitmproxy | 需要 | 功能全，可自动化 |
| 同 WiFi + 无 root | Reqable | 不需要 | 无 root 限制 |
| 手机开热点 | Reqable | 不需要 | 唯一可用的方案 |
