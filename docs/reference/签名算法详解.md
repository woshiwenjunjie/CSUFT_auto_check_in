## 1. 算法公式

### 1.1 FlySource-sign 请求头签名

```
sign = outer_md5 + "1." + base64(timestamp)

其中：

outer_md5 = MD5(path + "?sign=" + inner_md5)
inner_md5 = MD5(timestamp_str + token)
```

#### 参数说明

| 参数 | 类型 | 来源 | 说明 |
|------|------|------|------|
| path | string | URL 路径 | 去掉 ?query 部分，如 /api/.../getListForApp |
| timestamp | int | datetime.now() | 13 位毫秒 Unix 时间戳 |
| token | string | OAuth 响应 | access_token，未登录时为空字符串 "" |
| MD5 | -- | 标准算法 | 32 位十六进制小写 |
| Base64 | -- | 标准算法 | 对时间戳字符串的 UTF-8 字节编码 |
| "1." | string | 硬编码 | 固定分隔符，疑为签名版本号 |

#### 计算步骤示例

以 path="/api/test", timestamp=1700000000000, token="test_token" 为例：

```
Step 1: ts_str = "1700000000000"

Step 2: inner = MD5("1700000000000" + "test_token")
              = MD5("1700000000000test_token")
              = "91a1f8f522e5247a824fe78bf88b62a0"

Step 3: outer_str = "/api/test" + "?sign=" + "91a1f8f522e5247a824fe78bf88b62a0"
                  = "/api/test?sign=91a1f8f522e5247a824fe78bf88b62a0"

Step 4: outer = MD5("/api/test?sign=91a1f8f522e5247a824fe78bf88b62a0")
              = "611caada2eb45d1c20e53b2fda90fc18"

Step 5: b64 = Base64("1700000000000")
            = "MTcwMDAwMDAwMDAwMA=="

Step 6: sign = "611caada2eb45d1c20e53b2fda90fc18"
               + "1."
               + "MTcwMDAwMDAwMDAwMA=="
             = "611caada2eb45d1c20e53b2fda90fc181.MTcwMDAwMDAwMDAwMA=="
```

#### 签名在 HTTP 请求中的位置

```http
GET /api/flySource-yxgl/dormSignTask/getListForApp?current=1&size=10
Host: simp.csuft.edu.cn
Authorization: Basic Zmx5c291cmNl...
FlySource-Auth: eyJhbGciOiJSUzI1NiJ9...
FlySource-sign: 611caada2eb45d1c20e53b2fda90fc181.MTcwMDAwMDAwMDAwMA==
```

### 1.2 stuTaskId 提交校验哈希

打卡提交时，请求体中需要携带 stuTaskId 字段，用于服务器校验提交数据的完整性。

```
stuTaskId = MD5(JSON.stringify({
    latitude: "28.131085",
    longitude: "112.994644",
    locationAccuracy: "15.3",
    signDate: "2026-06-08",
    taskId: "b49ffb3790...",
    fileId: ""
}))
```

Key 顺序要求（与小程序源码对象字面量一致）：

```
latitude -> longitude -> locationAccuracy -> signDate -> taskId -> fileId
```

如果 Key 顺序不同，JSON.stringify 输出不同，MD5 结果不同，服务器校验失败。

## 2. 双语实现对照

### 2.1 MD5 与 Base64 底层函数

| 功能 | JavaScript（scripts/sign.js） | Python（src/utils/crypto.py） |
|------|-------------------------------|-------------------------------|
| MD5 | crypto.createHash('md5').update(s, 'utf8').digest('hex') | hashlib.md5(data.encode("utf-8")).hexdigest() |
| Base64 | Buffer.from(str, 'utf8').toString('base64') | base64.b64encode(data).decode("utf-8") |

两者等价。MD5 输出 32 位十六进制小写字符串。Base64 对 UTF-8 字节编码。

### 2.2 签名生成函数

| 步骤 | JavaScript | Python |
|------|-----------|--------|
| 时间戳转字符串 | String(timestamp) | str(timestamp) |
| 内层哈希 | md5(tsStr + token) | md5(ts_str + token) |
| 外层字符串 | path + '?sign=' + inner | path + "?sign=" + inner_hash |
| 外层哈希 | md5(outerStr) | md5(outer_str) |
| 时间戳 Base64 | b64(tsStr) | b64_encode(ts_str.encode("utf-8")) |
| 拼接 | outer + '1.' + b64(tsStr) | outer_hash + "1." + b64_encode(...) |

完整函数对比：

JavaScript（scripts/sign.js）：

```javascript
const crypto = require('crypto');

function md5(s) {
    return crypto.createHash('md5').update(s, 'utf8').digest('hex');
}

function b64(str) {
    return Buffer.from(str, 'utf8').toString('base64');
}

function generateSign(path, timestamp, token) {
    const tsStr = String(timestamp);
    const inner = md5(tsStr + token);
    const outer = md5(path + '?sign=' + inner);
    return outer + '1.' + b64(tsStr);
}
```

Python（src/utils/sign.py）：

```python
import hashlib, base64

def md5(data: str) -> str:
    return hashlib.md5(data.encode("utf-8")).hexdigest()

def b64_encode(data: bytes) -> str:
    return base64.b64encode(data).decode("utf-8")

def generate_sign(path: str, timestamp: int, token: str = "") -> str:
    ts_str = str(timestamp)
    inner_hash = md5(ts_str + token)
    outer_str = path + "?sign=" + inner_hash
    outer_hash = md5(outer_str)
    return outer_hash + "1." + b64_encode(ts_str.encode("utf-8"))
```

### 2.3 stuTaskId 构造

JavaScript（小程序源码）：

```javascript
var hashData = {
    latitude: res.latitude.toString(),
    longitude: res.longitude.toString(),
    locationAccuracy: this.stuSignData.locationAccuracy.toString(),
    signDate: this.signDate,
    taskId: this.params.taskId,
    fileId: ""
}
this.stuSignData.stuTaskId = md5(JSON.stringify(hashData))
```

Python：

```python
import json

hash_input = {
    "latitude": str(lat),
    "longitude": str(lng),
    "locationAccuracy": accuracy,
    "signDate": date,
    "taskId": task_id,
    "fileId": file_id or "",
}
stu_task_id = md5(
    json.dumps(hash_input, ensure_ascii=False, separators=(",", ":"))
)
```

关键细节：
- ensure_ascii=False -- 不转义非 ASCII 字符
- separators=(",", ":") -- 紧凑格式，对齐 JS JSON.stringify 默认输出
- 坐标值转为字符串（str(lat)）

### 2.4 Basic Auth 认证头

```
Authorization: Basic base64(ClientId + ":" + ClientSecret)
```

固定凭据（可被环境变量覆盖）：

| 属性 | 微信小程序版 | WebVPN 版 |
|------|-------------|-----------|
| ClientId | flysource_wise_wxapp | flysource_wise_app |
| ClientSecret | DA788asdUDjnasd_flysource_wxappdsdadDAIUiuwqe | DA788asdUDjnasd_flysource_dsdadDAIUiuwqe |

Python 实现：

```python
def generate_basic_auth(client_id: str, client_secret: str) -> str:
    raw = f"{client_id}:{client_secret}"
    return "Basic " + b64_encode(raw.encode("utf-8"))
```

## 3. 逐行推导：从 wxapkg 到 Python

### 3.1 定位签名代码

反编译 app-service.js（约 310KB）后，搜索 "FlySource-sign" 或 "SignHeader" 定位到请求拦截器。关键代码段（minified JS 还原）：

```javascript
var interceptor = {
    invoke: function(config) {
        var n = (new Date).getTime()        // timestamp
        var d = getToken()                   // token
        var r = config.url.split("?")[0]    // path
        r += "?sign="                         // signPath
        if (d) {
            config.header["FlySource-Auth"] = d
            config.header["FlySource-sign"] =
                md5(r + md5(n + d)) + "1." + gBase64.encode(n.toString())
        }
    }
}
```

### 3.2 逐行转译

| 行号 | 源码 | 含义 | Python 翻译 |
|------|------|------|------------|
| 1 | var n = (new Date).getTime() | 获取毫秒时间戳 | int(datetime.now(tz).timestamp() * 1000) |
| 2 | var d = getToken() | 从 storage 取 token | self.token（实例变量） |
| 3 | var r = config.url.split("?")[0] | 取 URL 路径，去 query | urlparse(url).path |
| 4 | r += "?sign=" | 拼接 sign 前缀 | path + "?sign=" |
| 5 | md5(n + d) | 内层 MD5：时间戳+token | md5(str(ts) + token) |
| 6 | md5(r + md5(...)) | 外层 MD5：signPath + 内层 | md5(outer_str) |
| 7 | +"1."+ | 固定分隔符 | + "1." + |
| 8 | gBase64.encode(n.toString()) | 时间戳 Base64 | b64_encode(str(ts).encode("utf-8")) |

### 3.3 关键发现

1. 时间戳是字符串拼接：JS 中 n + d 在 number + string 时自动转字符串，Python 需显式 str(timestamp) + token
2. "?sign=" 是 path 的一部分：外层 MD5 的输入是 path + "?sign=" + inner_hash，不是 path + inner_hash
3. "1." 是硬编码：小程序源码写死 +"1."+，没有版本协商机制
4. Base64 编码对象是时间戳的十进制字符串：如 "1700000000000" 的 UTF-8 字节，不是整数的二进制表示

## 5. 常见陷阱

### 陷阱 1：缺少微信环境头导致"签名错误"

服务器在校验签名之前先检查 Referer 和 User-Agent。缺少这些头时直接返回 {"msg": "签名错误"}，即使签名本身计算正确。

必须携带的头：

```python
{
    "Referer": "https://servicewechat.com/wx0e47c34c9982aa09/7/page-frame.html",
    "User-Agent": "Mozilla/5.0 ... MicroMessenger ... MiniProgramEnv/android",
    "charset": "utf-8",
}
```

排查顺序：先检查环境头是否完整，再怀疑签名算法。

### 陷阱 2：签名路径包含 query string

```
正确: path = "/api/flySource-yxgl/dormSignTask/getListForApp"
错误: path = "/api/flySource-yxgl/dormSignTask/getListForApp?current=1&size=10"
```

签名只使用 URL 的路径部分，不包括 query string。

### 陷阱 3：MD5 十六进制大小写

服务器期望小写十六进制。Python 的 hexdigest() 和 Node.js 的 digest('hex') 默认输出小写，不需要额外处理。如果使用其他语言，确保输出 32 位小写十六进制字符串。

### 陷阱 4：Base64 编码对象是时间戳字符串的 UTF-8 字节

```python
# 正确
b64_encode(str(timestamp).encode("utf-8"))

# 错误 -- 对整数的二进制表示做 Base64
b64_encode(timestamp.to_bytes(8, 'big'))
```

### 陷阱 5：stuTaskId 的 Key 顺序

json.dumps() 在 Python 3.7+ 中默认保持插入顺序，但必须确保字典的 Key 顺序与小程序源码一致。使用显式的字典字面量，不要从其他来源合并字典。

```python
# 正确：显式字面量，顺序固定
hash_input = {
    "latitude": ...,
    "longitude": ...,
    "locationAccuracy": ...,
    "signDate": ...,
    "taskId": ...,
    "fileId": ...,
}

# 错误：合并字典，顺序不可控
hash_input = {}
hash_input.update({"latitude": ...})
```

### 陷阱 6：时区不一致

开发环境（Windows 北京时间）和服务器环境（GitHub Actions UTC）的 datetime.now() 返回值不同。务必使用 timezone-aware datetime：

```python
from datetime import datetime, timezone, timedelta

BEIJING_TZ = timezone(timedelta(hours=8))
ts = int(datetime.now(BEIJING_TZ).timestamp() * 1000)
```

.timestamp() 方法将 aware datetime 转换为 UTC epoch 秒数，所以北京时间 21:05 的 epoch 值等于 UTC 13:05 的 epoch 值，正确反映了物理时间点。
