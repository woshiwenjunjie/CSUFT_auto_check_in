# 017 — SCF 部署重构与测试完善（2026-06-12）

## 背景

腾讯云 SCF 部署方案已存在，但代码存在多个问题：
- 变量名不够清晰（`client` → `api_client`, `cur_offset` → `current_offset_degrees`）
- 缺少顶层模块注释（流程图、设计原理、环境变量表）
- 模块级时间冻结风险（`NOW` 常量在 warm start 时不更新）
- 缺少自动化测试覆盖

## 改动

### 1. deploy/tencent-scf/ 全模块重构

**handler.py**：
- 新增顶层注释：执行流程图、异常安全网设计、事件结构说明
- 空 event 返回 `{"status": "healthy"}` 支持健康检查
- try/except 捕获所有异常并通知

**checkin.py**：
- 新增顶层注释：编排流程图、延迟时间函数原理、GPS 退避策略、5 种状态码
- 变量重命名：`client`→`api_client`、`dorm_lat`→`dormitory_lat`、`dorm_lng`→`dormitory_lng`、`cur_offset`→`current_offset_degrees`、`min_offset`→`min_offset_degrees`、`stu_task_id`→`student_task_id`、`src_dst`→`target_src_dir`
- 时间函数延迟化：`NOW`→`_now()`、`NOW_STR`→`_now_str()`、`DATE_STR`→`_date_str()`
- 配置合并为 `get_env_str(key, default)` 简化读取
- GPS 偏移退避：0.002° 起始 → 折半 → 最多 6 次 → 1e-6° 最小阈值

**notify.py**：
- 新增顶层注释：Server酱 推送流程、2 次重试 3s 间隔
- 新增 `SEND_TIMEOUT` 常量
- 环境变量统一 `get_env_str` 读取

**deploy.py**：
- 新增顶层注释：打包部署架构、环境变量表
- `src_dst`→`target_src_dir`、`pkg_path`→`package_path`
- 错误处理三明治（try/except/finally）
- 运行时版本变量化：`SCF_RUNTIME` 环境变量（默认 Python 3.12）
- `scf_bootstrap` 文件删除（标准 Python 运行时无需）
- `package.sh` 移除 `scf_bootstrap` 引用

**package.sh**：
- 移除 `scf_bootstrap` 和 `.env` 打包

## 关键教训
1. conftest 模块级 mock 的杀伤半径比想象大：`from notify import send_serverchan` 在 import 时绑定的是 MagicMock 而非真实函数
2. 采用不 mock conftest + `@patch('checkin.xxx')` 定点拦截模式，避免干扰无关测试
3. `_build_notification` 标题中的日期格式为 `"✅ CSUFT 打卡成功 · 2026-06-12"`，`^` 锚点断言不匹配，改用 `re.search(r"\d{4}-\d{2}-\d{2}", title)`
