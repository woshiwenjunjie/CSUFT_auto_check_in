## 1. request.js — 请求拦截器与签名算法

### 1.1 模块概览

`request.js` 是 uni-app 框架的 HTTP 拦截器，所有 `uni.request()` 调用都会经过它。它的核心职责是：**在每次请求发出前，自动注入认证头和签名**。

### 1.2 全局配置常量

```javascript
// 从 app-service.js pos 1391 附近提取
exports.BaseURL = "https://simp.csuft.edu.cn"
exports.ClientId = "flysource_wise_wxapp"
exports.ClientSecret = "DA788asdUDjnasd_flysource_wxappdsdadDAIUiuwqe"
exports.SignHeader = "FlySource-sign"      // 签名放在这个请求头
exports.TokenHeader = "FlySource-Auth"      // token 放在这个请求头
```

> 📌 **要点**：`ClientId` 和 `ClientSecret` 是小程序编译时写死的常量，不是从服务器获取的。每个学校有不同的 ClientId/Secret。它们用于生成 Basic Auth 头，本质上是客户端身份的"用户名密码"。

### 1.3 MD5 实现

```javascript
// 从 app-service.js 提取 — 标准 MD5 的微信小程序实现
function md5(s) {
    return crypto.createHash('md5').update(s, 'utf8').digest('hex')
}
```

- 使用微信内置的 `crypto` 模块（`crypto-js` 或微信基础库内建）
- 输出 32 位十六进制小写字符串
- **与我们 Python 的 `hashlib.md5(data.encode("utf-8")).hexdigest()` 完全等价**

### 1.4 Base64 实现

```javascript
// 从 app-service.js 提取 — 微信 gBase64 模块
gBase64.encode(str)  // 标准 Base64 编码
```

- 微信基础库内建，非 `btoa()`
- 与 Python `base64.b64encode(data).decode()` 完全等价
- 通过 `scripts/sign.js` 做 Node.js 交叉验证时也使用相同算法

### 1.5 请求拦截器核心逻辑（完整还原）

```javascript
// ═══════════════════════════════════════════════
// 以下是从 minified JS 中还原并重新格式化的代码
// 原始变量名: n=timestamp, d=token, r=pathWithSign, a=config
// ═══════════════════════════════════════════════

var interceptor = {
    invoke: function(config) {
        // 1. 获取当前毫秒时间戳
        var timestamp = (new Date).getTime()

        // 2. 读取已登录用户的 access_token
        var token = getToken()    // 从 wx.storage 读取

        // 3. 拼接签名用的 path（去掉 query string，加上 "?sign=" 前缀）
        var urlPath = config.url.split("?")[0]     // 如 "/api/flySource-yxgl/dormSignTask/getListForApp"
        var signPath = urlPath + "?sign="           // 如 "/api/.../getListForApp?sign="

        // 4. 如果 URL 不是完整 URL，拼接 BaseURL
        if (!config.url.startsWith("http")) {
            config.url = BaseURL + config.url
        }

        // 5. 设置超时
        config.timeout = 60000   // 60 秒

        // 6. 默认请求头
        if (isEmpty(config.header)) {
            config.header = { "Content-Type": "application/json" }
        }

        // 7. 注入 Basic Auth（除非接口标记 authorization=false）
        if (config.authorization !== false) {
            var authRaw = ClientId + ":" + ClientSecret
            var authEncoded = gBase64.encode(authRaw)
            config.header["Authorization"] = "Basic " + authEncoded
        }

        // 8. 如果有 token，注入 Token 和签名
        if (token) {
            config.header[TokenHeader] = token           // "FlySource-Auth"

            // ════════════════════════════════════════
            // 🔑 签名算法（单行完整公式）
            // ════════════════════════════════════════
            config.header[SignHeader] =
                md5(signPath + md5(timestamp + token))
                + "1."
                + gBase64.encode(timestamp.toString())

            // 拆解：
            //   inner  = md5(timestamp + token)
            //             ↑ timestamp 先转字符串再拼接 token
            //   outer  = md5(signPath + inner)
            //             ↑ signPath = url路径 + "?sign="
            //   result = outer + "1." + base64(timestamp)
            //             ↑ "1." 是固定分隔符（可能是版本号）
        }
    }
}

// 注册拦截器到 uni-app 框架
uni.addInterceptor("request", interceptor)
uni.addInterceptor("uploadFile", interceptor)  // 上传文件也走相同逻辑
```

### 1.6 签名公式图解

```
输入: path="/api/test", timestamp=1700000000000, token="abc123"

Step 1: signPath = "/api/test" + "?sign="
        → "/api/test?sign="

Step 2: inner = md5("1700000000000" + "abc123")
        → md5("1700000000000abc123")
        → "3f8c6b9a..."

Step 3: outer = md5("/api/test?sign=" + "3f8c6b9a...")
        → md5("/api/test?sign=3f8c6b9a...")
        → "a1b2c3d4..."

Step 4: tsB64 = base64("1700000000000")
        → "MTcwMDAwMDAwMDAwMA=="

Step 5: result = "a1b2c3d4..." + "1." + "MTcwMDAwMDAwMDAwMA=="
```

### 1.7 签名算法的几个关键细节

| 细节 | 源码证据 | Python 对应 |
|------|---------|------------|
| timestamp 是字符串拼接 | `md5(n+d)` — `n` 是 `(new Date).getTime()`（number），但 JS 的 `+` 在 number+string 时自动转字符串 | `md5(str(timestamp) + token)` ✅ |
| signPath 包含 `"?sign="` | `e.url.split("?")[0]+"?sign="` — 永远拼接 `?sign=` | `path + "?sign="` → `generate_sign()` 的 `outer_str` |
| inner 是字符串拼接，非相加 | `md5(n+d)` — JS 中 number + string = concat | `md5(ts_str + token)` |
| `"1."` 是固定分隔符 | 硬编码 `+"1."+` | 硬编码 `+ "1."` |
| Base64 是标准编码 | `gBase64.encode` | `base64.b64encode().decode()` ✅ |

### 1.8 请求出口函数

```javascript
// 实际被业务代码调用的 request 函数
exports.request = function(options) {
    return new Promise(function(resolve, reject) {
        uni.request(Object.assign({}, options, {
            success: function(res) {
                if (res.statusCode === 200) {
                    var data = res.data
                    // 🔴 401 检测：token 过期自动跳转登录页
                    if (data.code === 401) {
                        toLoginPage()           // 跳转到登录流程
                        reject(data)
                    } else {
                        resolve(data)           // 正常返回
                    }
                } else {
                    reject(res)
                }
            },
            fail: reject                        // 网络错误直接 reject
        }))
    })
}
```

> 📌 **要点**：`code === 401` 时自动跳登录页。这就是我们 Python `_token_expired()` 的判断依据 — `resp.get("code") == 401`。

## 2. authUtil.js — OAuth 认证模块

### 2.1 模块概览

`authUtil.js` 封装了飞源统一认证的所有 API 调用，包含 5 个方法。它们全部调用 `t.request()`（上述拦截器），所以**自动带有 Basic Auth 和签名**。

### 2.2 getCaptcha — 获取验证码

```javascript
// 密码登录的验证码获取
getCaptcha: function(type) {
    return request({
        url: "/api/flySource-auth/captcha",
        method: "GET",
        authorization: false,    // ← 🔑 不传 Basic Auth！此接口是公开的
        data: { type: type }     // 默认 "captcha"
    })
}
```

**对应路径**：`GET /api/flySource-auth/captcha?type=captcha`

> 📌 **为什么 `authorization: false`？** 小程序源码在此端点标记了 `authorization=false`，因为验证码接口不需要客户端认证。我们在 Python 中用 `no_auth=True` 参数实现了相同行为。

### 2.3 signIn — 密码登录

```javascript
signIn: function(tenantId, username, password, grantType, captchaKey, captchaCode) {
    return request({
        header: {
            "Content-Type": "application/x-www-form-urlencoded",
            "Tenant-Id": tenantId,              // 学校租户 ID
            "Captcha-Key": captchaKey,           // 从 getCaptcha 接口获取
            "Captcha-Code": captchaCode          // 用户输入的验证码
        },
        url: "/api/flySource-auth/oauth/token",
        method: "POST",
        data: {
            tenantId: tenantId,
            username: username,
            password: md5(password),              // ← 🔑 密码先 MD5 再传输！
            grant_type: grantType,                // "password"
            scope: "all"
        }
    })
}
```

**对应路径**：`POST /api/flySource-auth/oauth/token`（`grant_type=password`）

**关键发现：**
- 密码在客户端做 MD5 后传输 — 服务器存的是 MD5（密码的 MD5）
- `Captcha-Key` 和 `Captcha-Code` 从验证码接口获取
- 此函数在源码中**被定义了但从未被任何页面调用** — 是死代码！

### 2.4 signInOpenId — OpenID 登录（实际使用的登录方式）

```javascript
signInOpenId: function(tenantId, username, password, openid, bindState) {
    return request({
        header: {
            "Content-Type": "application/x-www-form-urlencoded",
            "Web-Type": "wxapp",            // ← 🔑 声明自己是微信小程序端
            "Tenant-Id": tenantId
        },
        url: "/api/flySource-auth/oauth/token",
        method: "POST",
        data: {
            tenantId: tenantId,
            username: username,              // 学号，bindState=0 时为空
            password: password !== "" ? md5(password) : "",  // ← MD5 或空
            grant_type: "wxapp",             // ← 不同于密码登录
            openid: openid,                  // ← 🔑 OpenID！
            bindState: bindState,            // 0=仅登录  1=绑定并登录
            scope: "all"
        }
    })
}
```

**对应路径**：`POST /api/flySource-auth/oauth/token`（`grant_type=wxapp`）

**bindState 的两种模式：**

| bindState | 场景 | username/password | 服务器行为 |
|-----------|------|-------------------|-----------|
| 0 | 小程序首页加载（仅登录） | 空字符串 | 用 OpenID 换取 token，不绑定学号 |
| 1 | 首次配置/绑定页面 | 填入学号和密码 | 将 OpenID 与学号绑定，同时返回 token |

### 3.4 GPS 定位获取（关键）

```javascript
function getLocationAndCheck() {
    wx.getLocation({
        type: "gcj02",                    // ← 🔑 国测局坐标系（GCJ-02），非 WGS-84！
        isHighAccuracy: true,             // 高精度模式
        highAccuracyExpireTime: 3100,     // 高精度有效期 3.1 秒
        success: function(res) {
            // 1. 记录当前坐标
            this.stuSignData.signLat = res.latitude
            this.stuSignData.signLng = res.longitude

            // 2. 计算当前坐标与宿舍坐标的距离
            this.stuSignData.locationAccuracy = getDistance(
                this.stuSignData.signLat,       // 当前纬度
                this.stuSignData.signLng,       // 当前经度
                this.location.locationLat,       // 宿舍纬度
                this.location.locationLng        // 宿舍经度
            )

            // 3. 校验是否在精度范围内
            this.checkStuSign =
                Number(this.stuSignData.locationAccuracy)  // 计算出的距离
                < Number(this.location.locationAccuracy)   // 任务设定的精度上限

            // 4. 构造 stuTaskId
            var hashData = {
                latitude: res.latitude.toString(),
                longitude: res.longitude.toString(),
                locationAccuracy: this.stuSignData.locationAccuracy.toString(),
                signDate: this.signDate,
                taskId: this.params.taskId,
                fileId: ""                       // 拍照上传后才有
            }
            this.stuSignData.stuTaskId = md5(JSON.stringify(hashData))

            // 5. 记录定位时间
            this.getLocation = new Date

            // 6. 延迟 1 秒展示结果（避免闪烁）
            setTimeout(function() {
                if (this.checkStuSign) {
                    uni.hideLoading()           // 在范围内，准备提交
                } else {
                    uni.showToast({
                        title: "不在打卡范围内！",
                        icon: "none"
                    })
                }
            }, 1000)
        },
        fail: function(err) {
            if (err.errMsg === "getLocation:fail auth deny") {
                // 用户拒绝了定位权限
                uni.showModal({
                    title: "需要定位权限",
                    content: "请在设置中允许小程序获取位置信息"
                })
            }
        }
    })
}
```

> 📌 **GCJ-02 坐标系**：微信 `wx.getLocation()` 默认返回 GCJ-02（火星坐标系），不是 GPS 原始坐标（WGS-84）。偏移量在 50-500 米。学校服务器存储的宿舍坐标也是 GCJ-02，所以不用担心坐标系不一致的问题。我们 Python 模拟时直接用宿舍坐标 + 小偏移即可。

### 3.5 距离计算函数（getDistance）

```javascript
function getDistance(lat1, lng1, lat2, lng2) {
    var radLat1 = lat1 * Math.PI / 180          // 纬度转弧度
    var radLat2 = lat2 * Math.PI / 180
    var deltaLat = radLat1 - radLat2             // 纬度差
    var deltaLng = lng1 * Math.PI / 180 - lng2 * Math.PI / 180  // 经度差

    var a = 2 * Math.asin(
        Math.sqrt(
            Math.pow(Math.sin(deltaLat / 2), 2)
            + Math.cos(radLat1) * Math.cos(radLat2)
            * Math.pow(Math.sin(deltaLng / 2), 2)
        )
    )

    a *= 6378.137                   // 地球半径（千米）
    return Math.round(10000 * a) / 10   // 转为米，四舍五入到整数
}
```

**对照 Python 实现：**

```python
def haversine(lat1, lng1, lat2, lng2):
    R = 6371000                     # 地球半径（米）— Python 用 6371km
    phi1, phi2 = math.radians(lat1), math.radians(lat2)
    dphi = math.radians(lat2 - lat1)
    dlambda = math.radians(lng2 - lng1)
    a = math.sin(dphi/2)**2 + math.cos(phi1)*math.cos(phi2)*math.sin(dlambda/2)**2
    return R * 2 * math.atan2(math.sqrt(a), math.sqrt(1-a))
```

> ⚠️ **微小差异**：JS 原版用地球半径 6378.137km（赤道半径），我们 Python 实现用 6371km（平均半径）。差异约 0.1%，对打卡校验（精度上限通常 >100m）没有影响。JS 版用 `Math.asin`，Python 用 `Math.atan2(sqrt(a), sqrt(1-a))` — 数学等价。

```javascript
    api.postUrl(url, this.stuSignData).then(function(res) {
        uni.hideLoading()

        if (res.success) {
            showSuccessDialog()       // 打卡成功弹窗 + 更新状态
        } else {
            uni.showToast({
                title: res.msg || "失败，请重试",
                icon: "none"
            })
        }
    }).catch(function(err) {
        uni.hideLoading()
        uni.showToast({
            title: err.msg || "提交失败",
            icon: "none"
        })
    })
}
```

### 3.7 stuTaskId 构造详解

```javascript
// 完整构造（不带照片）
var hashData = {
    latitude: "28.131085",            // 当前纬度（字符串）
    longitude: "112.994644",          // 当前经度（字符串）
    locationAccuracy: "15.3",        // 计算出的距离（字符串）
    signDate: "2026-06-08",          // 打卡日期
    taskId: "b49ffb3790...",         // 任务 ID
    fileId: ""                        // 文件 ID，未上传照片时为空字符串
}
var stuTaskId = md5(JSON.stringify(hashData))
// JSON.stringify 使用紧凑格式（无空格）
// → md5('{"latitude":"28.131085","longitude":"112.994644","locationAccuracy":"15.3","signDate":"2026-06-08","taskId":"b49ffb...","fileId":""}')
```

**Key 顺序要求**（从源码 `JSON.stringify` 的对象字面量推断）：
```
latitude → longitude → locationAccuracy → signDate → taskId → fileId
```

**Python 对应实现**（`_prepare_stu_sign_data`）：
```python
hash_input = {
    "latitude": str(cur_lat),
    "longitude": str(cur_lng),
    "locationAccuracy": loc_accuracy,
    "signDate": sign_date,
    "taskId": task_id,
    "fileId": file_id or "",
}
stu_data["stuTaskId"] = md5(
    json.dumps(hash_input, ensure_ascii=False, separators=(",", ":"))
)
```

> 📌 `separators=(",", ":")` 确保 JSON 紧凑格式（无空格），与 JS 的 `JSON.stringify()` 输出一致。

## 5. 源码中发现的设计细节

### 5.1 密码登录是死代码

`signIn()` 函数虽然在小程序源码中完整存在，但**从未被任何页面调用**。所有登录都走 `signInOpenId()` 路径。这意味着：
- 密码登录（`grant_type=password`）的 `Captcha-Key`/`Captcha-Code` 头在实际使用中永远不会被触发
- CLI 的 `login` 命令是"以防万一"的备用方案

### 5.2 `"1."` 分隔符的含义

签名中的 `"1."` 是飞源的签名版本号。未来如果更新签名算法，可能会变为 `"2."`。从这个角度看，签名公式更准确的理解是：

```
sign = MD5(path + "?sign=" + MD5(timestamp + token)) + <version> + Base64(timestamp)
```

### 5.3 坐标缓存 3 秒策略

打卡页面的 `getLocation` 缓存有效期为 `highAccuracyExpireTime: 3100`（3.1 秒）。如果用户在 3 秒内多次点击打卡，不会重新获取定位，直接用上一次的坐标。这个设计是为了防止短时间内的重复 GPS 请求。

### 5.4 小程序不传 Referer/User-Agent

这些环境头是**微信基础库自动附加的**，不在应用层 JavaScript 中显式设置。所以反编译源码中看不到 `Referer` 或 `User-Agent` 的设置代码。但服务器确实会校验这些头 — 我们的 CLI 请求必须手动添加。

### 5.5 接口路径分组规律

```
/api/flySource-auth/...        — 认证相关（OAuth2 token、captcha、logOut）
/api/flySource-base/...        — 基础数据（openApi、dataDictionary）
/api/flySource-yxgl/...        — 业务接口（dormSignTask、dormSignRecord）
/api/flySource-resource/...    — 文件/资源（oss/endpoint）
```

### 5.6 authorization=false 的接口

从源码中找到两个 `authorization: false` 的接口：
1. `getCaptcha` — 验证码接口（公开）
2. `signOut` — 退出登录（公开）

`getOpenidByJsCode` 没有 `authorization: false`，所以仍然会带 Basic Auth。
