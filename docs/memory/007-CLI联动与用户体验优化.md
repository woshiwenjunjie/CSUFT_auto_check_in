# 007 — CLI 联动与用户体验优化

日期：2026-06-07

## 改动概要

v0.7.0 解决签名问题后，对 CLI 工具做了全面的联动优化，目标是**减少重复输入、提升使用流畅度**。

## 改动清单

### 1. 密码输入显示占位符

`getpass.getpass()` 替换为自定义 `secure_input()`：

```python
def secure_input(prompt="密码: "):
    """密码输入，回显 * 占位符
    Windows: msvcrt.getch() → 逐字符回显 *
    Unix:    termios.setraw() → 逐字符回显 *
    Backspace / Ctrl+C 均有处理
    """
```

- 之前：输入密码时光标不动，不知道打了几个字
- 之后：`密码: ********` 清晰可见

### 2. 凭据持久化到配置文件

`~/.auto_check_in/config.json` 现在存储完整凭据：

```json
{
  "token": "eyJhbGciOiJI...",
  "tenant_id": "000000",
  "username": "2023****",
  "openid": "o************...",
  "password": "如果用了 --save-password",
  "task_id": "b49ffb3790..."
}
```

新增 CLI 参数：

| 参数 | 说明 |
|------|------|
| `login-openid --save-password` | 保存密码到配置文件 |
| `login-openid --force-input` | 跳过配置文件，强制交互输入 |
| `tasks --no-save` | 不自动保存任务 ID |

### 3. 命令联动（免输入任务 ID）

- `tasks` 执行后自动保存第一个任务的 ID 到 config
- `detail`/`checkin`/`record`/`month` 支持 `task_id` 留空，自动从 config 读取
- 如果 config 中没有 task_id 则提示先运行 `tasks`

```
# 完整联动流程
login-openid o6zFv5... 2023**** --save-password  # 首次：存入所有凭据
tasks                                               # 自动记住任务ID
detail                                              # 自动用已存ID
checkin                                             # 一键打卡
record                                              # 确认状态
```

### 4. 密码登录路径修复

对比 JS 源码发现 `sign_in` 缺少请求头，已修复：

- `get_captcha`：新增 `no_auth=True`（此端点在小程序中 `authorization=false`）
- `sign_in`：新增 `Tenant-Id`/`Captcha-Key`/`Captcha-Code` 请求头
- `_headers`：新增 `no_auth` 参数，支持跳过 Basic Auth

### 5. 任务 ID 解析统一入口

新增 `_resolve_task_id(args, cfg)` 函数，所有需要 task_id 的命令统一调用，逻辑集中在一点，方便维护。
